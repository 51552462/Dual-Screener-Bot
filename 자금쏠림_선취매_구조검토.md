## 1. 장세별(Regime) 자금 쏠림 추적 현황

### 1-1. 자금 쏠림 수치화 — 3개 독립 레이더

**(A) KR 외인·기관 실순매수 다이버전스 (`smart_money_tracker.py`)**
- pykrx 투자자별 순매수 상위(외국인+기관합계, KOSPI/KOSDAQ) 합산 → 종목별 순매수 거래대금(`krw`)·거래량(`vol`) 누적(`_try_pykrx_flow_leaderboard`, 실패 시 네이버 iframe 폴백).
- **쏠림 = 순매수 폭증 + 가격 미반응(다이버전스)**. 최근 5일 가격변화 `> +1.8%` 면 제외(`smart_money_tracker.py:537-539`) → "기관은 사는데 가격은 안 오른 = 매집" 만 통과.
- 평단 추정: 일별 순매수량 가중 **1차원 칼만 필터**(`_kalman_latent_avg_cost_1d`) — 대량 순매수일 관측분산 R_t 축소(연속 매집일 streak 가중)로 평단이 체결가에 빠르게 수렴.
- 점수 `divergence_score = flat_component(가격눌림) + flow_component(log10 순매수대금×7.5, 상한45) + day_bonus`. → `SMART_MONEY_RADAR.picks`.

**(B) US 다크풀 프록시 (`smart_money_tracker.py:run_us_dark_pool_proxy`)**
- 13F·유료 피드 없이 **"저변동성 거래량 폭증"** 을 기관 흡수 매집으로 역추적.
- 조건(`_detect_dark_pool_accumulation`): `거래량 ≥ 20일평균×2.0`(`US_VOLUME_SURGE_MULT`) **AND** `|당일 등락률| ≤ 2.0%`(`US_LOW_VOLATILITY_ABS_PCT`). → 가격 안 움직이며 물량 흡수 = 신선한 매집 캔들.
- 점수 `surge_component(ratio×12, 상한60) + flat_component(저변동성 가점)`. → `SMART_MONEY_RADAR_US`.

**(C) 섹터 단위 쏠림 (`sector_rotation_store.py`)**
- `forward_trades` 진입을 일별 집계 → `dominant_sector_for_series`로 **당일 주도 섹터**(`sector_daily_leader`) 산출.
- 섹터 체류일(streak), 전이 이벤트(`sector_transition_event`), 7/30/90일 롤업(`sector_transition_rollup`)으로 "자금이 어디 머물고 어디로 이동 중인가"를 정량화.

### 1-2. 장세(BULL/BEAR)에 따라 임계값을 다르게 적용하는가?

**→ 쏠림 감지 임계값 자체는 장세 무관(단일 기준)이다.**

- `US_VOLUME_SURGE_MULT=2.0`, `US_LOW_VOLATILITY_ABS_PCT=2.0`, KR 다이버전스 `+1.8%` 컷 등은 **하드코딩 상수**로, 현재 BULL/BEAR 여부에 따라 바뀌지 않는다(`smart_money_tracker.py:623-627`).
- 섹터 dominance/transition 집계도 장세 분기가 없다.

**단, "장세 의존성"은 쏠림 감지가 아니라 그 쏠림을 매매로 옮기는 단계(선취매)에서 별도 계층으로 존재한다:**
- `regime_analog_engine.frontrun_gate` — 현재 국면 벡터(SPX/KOSPI MA20 이격, VIX_z, PRI_z, macro_z, range5d)를 과거 에피소드와 **마할라노비스+DTW 앙상블**로 비교해 `REGIME_ANALOG_SCORE` 산출. 과거 에피소드는 `front_run_favorable` 플래그로 분류(V_RECOVERY·MASSIVE_BULL=유리, EXTREME_CRASH·CHOPPY_STAGNANT=불리).
- 즉 **"쏠림은 장세 무관하게 감지하되, 그 쏠림을 선취매로 베팅할지는 장세 유사도가 결정"** 하는 2단 구조다.

> 보강 포인트: 매집 임계값(거래량 배수·다이버전스 컷)을 BEAR에서 더 보수적으로(예: surge_mult↑) 두는 regime-adaptive 튜닝은 현재 없음.

## 2. 선취매(Pre-emption) 로직의 실체

### 2-1. 두 개의 예측 소스

**(가) 순환매 Markov 예측 → `PREDICTED_NEXT_SECTOR_{market}`**
- `sector_rotation_store.predict_next_sector_markov`: `sector_transition_rollup`으로 전이행렬 M 구성 → **2-step 전이(M²)** 의 argmax로 "현재 주도 섹터 다음에 올 섹터" 예측. confidence = `min(0.95, 0.4+0.08×top_count)`.
- `refresh_predicted_sector_via_store`가 `cfg["PREDICTED_NEXT_SECTOR_{mkt}"]`에 저장(+ `_FROM`, `_CONFIDENCE`, `_LAST_GOOD`).

**(나) 미국→한국 스필오버 → `US_SPILLOVER_SECTOR`**
- `sector_spillover_refresh.py`: 최근 7일 미국장 **고MFE 섹터**(승자 섹터) 그룹의 최빈 섹터를 `US_SPILLOVER_SECTOR`로 저장(`:120-126`).
- 표본 0일 때 `zero_sample_spillover.apply_zero_sample_spillover`가 US 다크풀(매집 징후) 기반으로 대체 산출 → `US_ZERO_SAMPLE_SPILLOVER` + `US_SPILLOVER_SECTOR`.
- `cross_market_ssot`가 US 섹터 → KR 논리 섹터 매핑(`kr_stock_matches_spillover`)을 제공.

### 2-2. '다음 주도주를 미리 사는' 논리 흐름

스캐너 진입 단(`forward/shared.py` `try_add_virtual_position` 내부)에서:

1. **예측·스필오버 로드** (`forward/shared.py:1921-1924`)
```1921:1924:forward/shared.py
            predicted_sector = sys_config.get(f"PREDICTED_NEXT_SECTOR_{market}", "NONE")
            is_rotation_prebuy = (sector == predicted_sector) and (sector != "기타/혼합")
            spillover_sector = str(sys_config.get("US_SPILLOVER_SECTOR", "NONE"))
            is_spillover_prebuy = (market == 'KR') and (sector == spillover_sector) and (sector != "기타/혼합")
```

2. **국면 유사도 안전 게이트** (`forward/shared.py:1935-1943`) — `regime_analog_engine.frontrun_gate`가 현재 국면이 "선취매 유리 과거 국면"과 `REGIME_ANALOG_FRONTRUN_MIN_SCORE`(기본 0.80) 이상 유사할 때만 `frontrun_allowed=True`. 불일치 시 `frontrun_blocked` → 선취매 특권 차단.
```1941:1943:forward/shared.py
            rotation_prebuy_active = is_rotation_prebuy and frontrun_allowed
            spillover_prebuy_active = is_spillover_prebuy and frontrun_allowed
            frontrun_blocked = (is_rotation_prebuy or is_spillover_prebuy) and not frontrun_allowed
```

3. **컷오프 완화(미리 사기 쉽게)** (`forward/shared.py:1945-1953`)
   - 순환매 선취매: 코사인·ML 컷오프 **×0.85**(15% 완화).
   - 스필오버 선취매: **×0.90**(10% 완화).

4. **태그 박제 + 켈리 증액** (`forward/shared.py:1971-2103`)
   - 통과 시 `sig_type`에 `[순환매 컷오프 특권 패스]` / `[🌐스필오버 선취매]` / `#순환매_선취매` 부착.
   - `#순환매_선취매` & `ROTATION_ADVANTAGE_ACTIVE=True` → **켈리 ×2.0**.
   - 스필오버 선취매 → **켈리 ×1.5**.
   - 게이트 차단 시 `[🔒선취매보류:국면불일치]` 태그만 남고 특권 없음.

> 별도로 `scanner_synergy_engine.per_ticker_scan_adjustments`도 동일 사상으로 `ROTATION_PRE`(cos/ml ×0.85, +3pt), `SPILLOVER_PRE`(×0.90, +4pt), `MACRO_SYNERGY` 보너스를 부여(스캐너 시너지 계층).

**요약 흐름도**
```
[US 고MFE 섹터/다크풀] → US_SPILLOVER_SECTOR ┐
[섹터 전이 Markov M²]  → PREDICTED_NEXT_SECTOR ┤
                                              ├─ 종목 섹터 일치? 
[국면벡터 마할라노비스+DTW] → frontrun_gate ──┘     │
                                                   ▼ (게이트 통과 시)
                        컷오프 완화(×0.85/0.90) + 켈리 증액(×2.0/1.5) + 태그 박제
```

## 3. 예측 적중률(Hit Rate)과 데스매치 피드백 구조

### 3-1. 적중률 추적 — 존재함 (섹터 예측 레벨)

`sector_rotation_store.record_prediction_outcome`이 **전일 예측 vs 실제 주도 섹터**를 비교해 적중을 기록:
- `sector_rotation_prediction_log` 테이블에 `predicted_sector / actual_sector / hit(1/0) / confidence_before / miss_streak_after` 적재.
- `sector_rotation_smoothing.apply_prediction_miss_smoothing`로 `ema_accuracy`·`miss_streak` 갱신, miss_streak 누적 시 `suggest_disable_rotation_advantage` → `ROTATION_ADVANTAGE_ACTIVE=False`로 자동 강등(`sector_rotation_store.py:615-618`).
- → "예측 섹터가 실제로 다음에 주도했는가"의 **명시적 hit rate 루프**가 있다.

### 3-2. PnL 우위 검증 — 존재함 (`system_auto_pilot.py` 엔진 12)

선취매로 **실제 돈을 벌었는지**를 일반 매매와 분리해 검증하는 핵심 피드백:
```1580:1602:system_auto_pilot.py
    rot_df = df[df['sig_type'].str.contains('#순환매_선취매', na=False)]
    std_df = df[~df['sig_type'].str.contains('#순환매_선취매', na=False)]
    ...
    if len(rot_df) >= 3:
        rot_pf = get_pf(rot_df)
        std_pf = get_pf(std_df)
        ...
        if rot_pf > std_pf * 1.5:
            current_config["ROTATION_ADVANTAGE_ACTIVE"] = True   # → 다음 주 켈리 2배
        else:
            current_config["ROTATION_ADVANTAGE_ACTIVE"] = False
```
- `#순환매_선취매` 태그 보유 거래의 **PF(Profit Factor)** 를 일반 그룹과 비교, **1.5배 우위** 증명 시에만 다음 주 켈리 2배 특권 부여(표본 ≥3건 가드). → 선취매 로직 **자체의 가중치를 PnL로 평가하는 폐루프**.

### 3-3. Deathmatch 본체는 선취매를 분리 랭킹하지 않음

- `evolution/regime_logic_crossmatrix.py`는 `groupby(["entry_regime","group_key"])`로 **장세별 × 로직별** 성과를 랭킹하지만, group_key가 `ledger_group_key`로 산출됨:
```36:44:evolution/deathmatch_battle_royale.py
def ledger_group_key(sig: str) -> str:
    raw = str(sig or "")
    ...
    s = re.sub(r"^\[.*?\]\s*", "", s)
    return (s.split(" [")[0].strip() or "UNKNOWN")
```
- 이 함수는 **선행 `[...]` 제거 + 첫 " [" 앞까지** 만 취하므로, 끝에 붙은 `#순환매_선취매`·`[🌐스필오버 선취매]` 태그가 **그룹키에서 제거**된다. 즉 데스매치 랭킹에서는 선취매 종목이 모(母) 전략에 합산될 뿐 **별도 arm으로 분리되지 않는다.**
- 따라서 "선취매 vs 일반"의 분리 평가는 **오직 엔진 12(§3-2)** 와 **섹터 예측 로그(§3-1)** 두 곳에서만 이뤄진다.

### 3-4. 피드백 루프 요약

```
선취매 진입(#순환매_선취매 태그)
   │
   ├─[섹터 예측 적중률] sector_rotation_prediction_log: hit/miss·EMA·miss_streak
   │        └─ miss 누적 → ROTATION_ADVANTAGE_ACTIVE=False (자동 강등)

   [국면 유사도] regime_analog frontrun_gate: 실패 국면 RL 페널티(penalize_episode)
```

## 4. 종합 진단 & 보강 여지 (관찰 기반, 변경 없음)

1. **쏠림 감지의 장세 무차별성** — 매집 임계값(거래량×2.0, 다이버전스 ±1.8/2.0%)이 BULL/BEAR 동일. BEAR에서 가짜 매집(데드캣) 필터를 위해 regime-adaptive 임계값 도입 여지.
2. **스필오버 선취매엔 PnL 폐루프 부재** — `#순환매_선취매`는 엔진12로 PF 검증되지만, **스필오버 선취매(켈리 ×1.5)** 는 동등한 별도 PF 우위 검증 루프가 없음(섹터 예측 로그와도 분리). 스필오버 태그 그룹의 PF 추적 추가 여지.
3. **Deathmatch 미분리** — 선취매 태그가 group_key에서 제거돼 데스매치 리더보드에서 모전략과 섞임. 선취매를 별도 arm으로 분리 랭킹하면 "선취매 알파"의 장세별 기여를 직접 가시화 가능.
4. **적중률↔PnL 연결** — 섹터 예측 hit rate와 실제 선취매 PnL이 별도 추적됨. 둘을 결합한 "예측 적중 시 평균 초과수익" 지표는 아직 없음.

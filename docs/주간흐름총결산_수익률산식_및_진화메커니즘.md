# 주간 Flow 총결산 — "주간 탑 수익률" 산식 & 진화 메커니즘 (코드 기반 전수 해설)

> 대상: `🗺️ [V100.0 퀀트 팩토리 주간 흐름(Flow) 총결산]` 의 `[N 일주일 자금·섹터 흐름]`
> 블록에 찍히는 **날짜 · 수익률(손익) · 승률 · 건수**가 (a) 한 건에서 나온 값인지,
> (b) 건수 평균인지, (c) 건수 총합인지, 그리고 (d) 이 수익률이 앞으로 어떻게 진화·발전하는지.

---

## 0. 한 줄 결론

- `🗓️ [일자별 실현 손익·승률]` 과 `💰 주간 누적` 에 찍히는 **손익 숫자는 "단건"도 "평균"도 아니라,
  그 구간(그 날짜 / 그 주)에 청산된 N건의 실현손익을 전부 더한 총합(SUM)** 이다.
- **승률** = 그 N건 중 수익난 건수의 비율(건수 기반), **건** = 청산 건수(N).
- 단, 블록마다 **합 / 평균 / 단건**이 섞여 있다(아래 §2 표가 정확한 분류). 특히 "수익률"이
  **금액(원/$)** 인지 **퍼센트(%)** 인지에 따라 의미가 달라진다.

---

## 1. 데이터 출발점 — 무엇을, 어떻게 1건으로 만드나

집계 모수 (`weekly_flow_report._load_week_closed_df`):
- `forward_trades` 중 `status LIKE 'CLOSED%'`, **최근 7일** 윈도(`closed_event_dates` + `in_date_window`),
  `INCUBATOR` 태그 제외, `final_ret` 결측 제거.
- 기준일은 **`exit_date`(청산일)** — 진입일이 아니다.

1건의 실현손익(금액) — `weekly_flow_pnl.row_realized_pnl`:
```
1건 실현손익 = notional × (final_ret% / 100)
notional(투입 노출액) = sim_kelly_invest → invest_amount → 시장 폴백(KR/US 40만)
```
- `final_ret` = **1건의 청산 수익률(%)**.
- 투입액 데이터가 없으면 **40만원/$ 폴백 노출**로 가정해 금액을 만든다(= 실제 투입이 아닌 가정 손익).

## 2. 블록별 숫자 산식 — 합 / 평균 / 단건 정확 분류

| 블록 | 표시 항목 | 산식 | 분류 |
|---|---|---|---|
| 🗓️ **일자별 실현 손익·승률** | 날짜 · 손익(금액) · 승률 · 건 | 손익 = **Σ**(그 날 1건손익) · 승률 = 수익건/총건 · 건 = 총건 | **총합(SUM)** + 건수기반 비율 |
| 💰 **주간 누적** | 손익(금액) · 승률 · 건 | 주간 전체 **Σ** · 승률 = 주간수익건/주간총건 | **총합** |
| 🏷️ **주도 섹터 Top2** | 섹터 · 손익(금액) · 건 · WR | 섹터별 **Σ**손익 · WR = 섹터수익건/섹터건 | **총합** |
| 🏆 **MVP 시그널 엔진** | 시그널 · 손익(금액) · 건 · 견인 | 시그널코어별 **Σ**손익 · 견인 = 대표종목 **Σ**손익 | **총합** |
| 📊 **태그 롤업** | `tag(n, cum=±%, WR=%)` | cum = **Σ**(1건 `final_ret%`) · WR = 수익건/n · (내부 mean = cum/n) | **수익률 % 합** |
| 🧬 **DNA 명예의전당/오답노트** | 종목(±%) | `nlargest(1)` / `nsmallest(1)` | **단건(SINGLE) 최고/최저** |
| ⚙️ **주간 청산 누적(가상·국고)** | 누적 손익(금액) | **Σ** | **총합** |

### 사용자가 본 그 숫자 = 🗓️ 일자별 블록
`weekly_flow_report.build_weekly_market_snapshot`:
```python
for exit_date, g in wdf.groupby("exit_date"):
    d_pnl = float(g["_realized_pnl"].sum())   # ← 그 날 N건 손익의 총합(SUM)
    wins  = int((g["final_ret"] > 0).sum())   # ← 수익난 건수
    total = int(len(g))                        # ← 청산 건수 N
# 승률 = wins / total * 100
```
→ **날짜에 찍힌 손익 = 그 날 청산된 건수(N)만큼의 실현손익을 전부 합산한 총합**,
   **승률 = 그 N건 중 수익 비율**, **건 = N**. (단건도 평균도 아님)

### "수익률"이 금액인지 %인지 주의
- 일자별 · 주간 누적 · 섹터 · MVP = **금액(원/$) 총합**.
- 태그 롤업 `cum=±%` = **퍼센트 단순합**(복리 아님; `forward_flow_tag_deep_dive`의 `cum_ret=("_fr","sum")`).
- DNA 명예의전당/오답노트 `(±%)` = **단일 베스트/워스트 트레이드 1건의 수익률**.

## 3. 그래서 이 수익률이 "앞으로 어떻게 진화·발전" 하나 (닫힌 피드백 루프)

이번 주 수치는 다음 주 손익에 **직접 곱해지지 않는다.** 대신 통계(승률·PnL·태그·DNA)가
**메타거버너·튜닝 파라미터를 갱신** → 다음 주 진입 종목·베팅 사이즈·필터가 바뀜 → 수익률이 진화한다.

```
[이번 주 CLOSED 7일 표본]
   │  (집계)
   ├─ 일자별/주간 Σ손익 · 승률 · 건
   ├─ 섹터 Σ · MVP 시그널 Σ
   ├─ 태그 롤업: cum%·WR·PF·독성태그
   └─ DNA 롤업: 지배 피처·명예/오답
        │  (해석 → 다음 주 규칙)
        ▼
  WeeklyActionPlan (weekly_action_plan.build_weekly_action_plan)
   - 주도 섹터 가중 유지·감시
   - 독성 태그 → 페널티(×0.85 등)·회피 가드 (FLOW_TAG_PENALTY_MULT)
   - Universal DNA 지배 축 → 캐리 필터 반영
   - 국면 · 유효 켈리(base × Meta mult) 표기
        │
        ├─ persist_weekly_baseline → WEEKLY_REPORT_BASELINE 저장
        │     └ 다음 주 Δ 비교: 코사인 허들 / ML박스 허들 / 유효 켈리 (상향 ➔ 하향 화살표)
        ▼
  MetaGovernor SSOT (다음 주 실제 거래에 작용)
   - DYNAMIC_SUPERNOVA_CUTOFF (초신성 코사인 허들)
   - DYNAMIC_ML_BOX_CUTOFF (ML박스 허들)
   - 유효 켈리 risk = base_dynamic_kelly_risk × meta_global_kelly_mult
        │
        ▼
[다음 주 스캐너] 진입 점수 컷·베팅 사이즈·태그 페널티가 갱신된 채로 거래
        │
        ▼
[다음 주 CLOSED 표본] → 다시 위로 (진화 반복)
```

핵심 진화 채널 (코드):
1. **독성 태그 페널티** — `_pick_toxic` 로 cum% 최저 태그 선정 → `FLOW_TAG_PENALTY_MULT`(기본 ×0.85)
   페널티·회피 가드 → 다음 주 해당 태그 종목 점수 감점.
2. **섹터 DNA 가중** — 주도 섹터(Top2)·DNA 지배 피처를 다음 주 캐리 필터/가중에 반영.
3. **켈리·허들 Δ 추적** — `WEEKLY_REPORT_BASELINE` 와 비교해 코사인/ML박스 허들·유효 켈리의
   주간 변화를 화살표로 노출 → 베팅 사이즈·진입 컷이 국면 따라 조정.
4. **라이프사이클** — COOLED 전략 벤치 대기/재기동 관측, 레지스트리(LIVE/COOLED/CANDIDATE) 운용.

## 4. 오해 포인트 · 주의

- **폴백 40만 notional:** 투입 데이터가 없는 건은 40만 노출로 가정 → 금액 합은 "가정 손익"(실투입 아님).
  → 금액의 절대치보다 **부호·승률·추세**로 해석할 것.
- **태그 cum% 단순합:** 표본 수(n)가 다르면 비교 왜곡 → `mean_ret`(평균)·`WR`·`PF` 를 함께 봐야 한다.
- **DNA 명예의전당/오답은 단건:** 대표성이 낮아, 블록은 대박·참사 각 **5건 이상**일 때만 의미 있다고 명시.
- **청산일(exit_date) 기준 7일 윈도:** 진입일 기준이 아니다. 늦게 청산되면 다음 주 표본으로 넘어간다.
- **INCUBATOR 제외:** 인큐베이터 트레이드는 총결산 손익·승률에서 빠진다.

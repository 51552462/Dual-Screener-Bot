# 스마트머니(Smart Money) — 데이터 없음 원인 & 퀀트 구조 기여 전수 정리

> 대상: 일일 통합 9분할 성과 리포트의 위성 브리핑 `[0/9]` 영역에서 KR/US 모두
> "스마트머니: 데이터 없음"으로 표시되는 문제와, 스마트머니가 실제로 퀀트 엔진에
> 어떻게 기여하는지에 대한 코드 기반(SSOT) 전수 감사.

---

## 0. 결론 요약 (TL;DR)

| 시장 | 표시 | 진짜 원인 | 성격 |
|---|---|---|---|
| **US** | `▪ 스마트머니(US): 데이터 없음 — KR pykrx 파이프 전용 (US 위성 P2 예정)` | **설계상 미구현 플레이스홀더.** US용 스마트머니(13F·기관 순매수) 데이터 소스가 아예 없음 | 버그 아님 (기능 미개발) |
| **KR** | `▪ 스마트머니: 데이터 없음` | `system_config.json`의 `SMART_MONEY_RADAR.picks`가 **비어 있음**. 이를 채우는 `smart_money_tracker.py`(KST 16:10 잡)가 **factory 크론에 없고 legacy `--daemon` 루프에만** 존재 → 데몬이 죽었거나 그날 픽이 0건이면 영구 공백 | 운영/이관 누락 |

- **US는 정상적인 "미구현" 상태**다. KR pykrx(외국인·기관 순매수) 파이프만 존재하고, US 위성(P2)은 코드에 자리만 잡혀 있다.
- **KR이 핵심 문제**다. 데이터 자체는 만들 수 있는데, 만드는 잡이 스케줄에서 빠져 있다. 이는 지난 OHLCV bulk(07:00) 누락과 **동일 계열의 이관 사고**다 — "항상-기동 `system_auto_pilot --daemon` 루프"가 시간 기반으로 돌리던 위성 잡(16:10 스마트머니, 15:40/16:20 상한가 부검, 17:00 둠스데이)이 cron SSOT로 일급 이관되지 않았다.

## 1. 스마트머니란 무엇이고 어떻게 만들어지나 (산출 파이프라인)

산출 SSOT: **`smart_money_tracker.py` → `system_config.json['SMART_MONEY_RADAR']`**

### 1.1 데이터 소스 (KR 전용)
- 1순위: **pykrx** 외국인·기관 순매수 상위 랭킹(`_try_pykrx_flow_leaderboard`)
- 폴백: **네이버 순매수 iframe**(`_naver_aggregate_flow`, 외인+기관 상위만)
- "2천 종 전수 루프 금지" — 상위 랭킹만 소수 요청으로 수집

### 1.2 후보 선별 로직 (`run_smart_money_tracker`, `smart_money_tracker.py:467`)
1. 순매수 금액(`krw`) 상위 **55종**만 후보.
2. 각 후보의 최근 OHLCV로 **가격 다이버전스 필터**: 최근 5일 종가 변화율 `price_change_pct > 1.8%`면 **제외**
   → "기관은 샀는데 가격은 아직 안 뜬"(매집·잠복) 종목만 남김.
3. 평단(`avg_price`) 추정:
   - pykrx 일별 순매수량+종가 시계열에 대해 **1차원 상태공간 칼만 필터**(`_kalman_latent_avg_cost_1d`)로 잠재 평균단가 추정.
     관측잡음 `R_t`를 일별 순매수 규모에 반비례시켜 **대량 매수일에 더 큰 신뢰**를 부여.
   - 칼만 실패 시 순매수 **VWAP**(`_flow_vwap`)로 폴백.
4. `divergence_score = flat_component + flow_component(+day_bonus)`:
   - `flat_component = max(0, 2.5 - |price_change_pct|)` (덜 올랐을수록 가점)
   - `flow_component = min(45, log10(krw+1) * 7.5)` (순매수 규모 가점)

### 1.3 산출 스키마 (고정)
```json
"SMART_MONEY_RADAR": {
  "updated_at": "YYYY-MM-DD HH:MM",
  "picks": {
    "<종목코드>": { "name": "...", "avg_price": 12345, "divergence_score": 41.2 }
  }
}
```
> **중요:** `if smart_picks:` 가 True일 때만 저장한다(`smart_money_tracker.py:556`).
> 즉 **픽이 0건이면 기존 값을 덮어쓰지 않고 그대로 둔다** → 한 번도 성공 못 했으면 키 자체가 없고, 리포트는 "데이터 없음"으로 표시.

### 1.4 스케줄 (문제 지점)
- 트리거: `system_auto_pilot.py:1983` — `--daemon` 루프 안 `now.hour == 16 and now.minute == 10`에서
  `_spawn_py_script("smart_money_tracker.py", "smart_money_1610")`.
- **cron(`deploy/factory.kr.crontab.example`)·factory 파이프라인(`factory_pipelines.py`) 어디에도 없음.**
- 즉 이 잡은 **`dante-factory`(=`system_auto_pilot --daemon`, `deploy/entrypoints/run_factory_daemon.sh`) 데몬이 살아 있을 때만** 16:10에 발화한다.

## 2. "데이터 없음"이 뜨는 정확한 코드 경로

리포트 위성 브리핑은 `satellite_intel_brief.py`가 생성한다.

### 2.1 KR (`_provider_smart_money_kr`, `satellite_intel_brief.py:394`)
```python
radar = ctx.cfg.get("SMART_MONEY_RADAR") or {}
picks = radar.get("picks", {}) ...
if len(picks) == 0:
    return [SatelliteLine("KR", f"▪ 스마트머니: 데이터 없음{updated}", priority=10)]
```
→ `picks`가 비면 무조건 "데이터 없음". picks가 비는 경우:
1. **데몬 미가동/크래시**: `dante-factory --daemon` 루프가 죽어 16:10 잡이 발화 안 됨.
   (과거에 발견된 `async_telegram_daemon.py`·`doomsday_bridge.py`·`report_pipeline_hydrate.py` 누락은
   이 루프를 import 단계에서 죽일 수 있어 **모든 위성 잡(16:10 포함)을 동반 중단**시킨다.)
2. **그날 픽 0건**: pykrx/네이버 차단 또는 다이버전스 필터(+1.8%)에 전부 걸러져 `smart_picks`가 빔
   → 저장 스킵(`if smart_picks:`) → 라다 공백 유지.
3. **최초 1회도 성공 못 함**: 키가 아예 없음.

### 2.2 US (`_provider_smart_money_us_placeholder`, `satellite_intel_brief.py:478`)
```python
def _provider_smart_money_us_placeholder(ctx):
    if ctx.report_market != "US":

        "▪ 스마트머니(US): 데이터 없음 — KR pykrx 파이프 전용 (US 위성 P2 예정)",

```
→ **하드코딩된 고정 문구.** US는 데이터 소스가 없으므로 항상 "데이터 없음". 런타임 실패가 아니라 미구현.

## 3. 스마트머니가 퀀트 구조에 기여하는 모든 경로 (소비처 전수)

스마트머니는 단순 표시용이 아니라 **진입 게이트·점수 가중·진화 학습·성과 검증**에 다층으로 물려 있다.

### (1) 신규 진입 교차검증 태그 — 모든 검색기 공통 관문
`try_add_virtual_position`(`forward/shared.py:1494`)는 모든 스캐너가 가상매매 편입 시 거치는 단일 관문이다.
그 안의 "통합 방어막"(`forward/shared.py:1566`)에서:
```python
avg_price = get_smart_money_avg_price_from_ssot(pre_sys_config, code_str)   # forward/shared.py:1453
if avg_price > 0 and abs(ep - avg_price)/avg_price <= 0.03:
    sig_type += " [🕵️세력매집_교차검증]"
```
- **의미:** 진입가(`ep`)가 스마트머니 추정 평단의 **±3% 이내**면 "세력 매집 구간에서 같이 진입"으로 보고
  시그널에 **교차검증 배지**를 부착한다. (하드 차단이 아니라 **신뢰 등급 승격** — 둠스데이/오답노트는 차단, 스마트머니는 가점성 태그.)
- **데이터 없음 영향:** `avg_price`가 항상 0 → 이 배지가 **한 번도 안 붙음** → 아래 (3) 버프 표본도 안 쌓임.

### (2) 평단 SSOT — 스나이퍼/평단 교차검증
`supernova_hunter.py:1945`:
```python
# SSOT: SMART_MONEY_RADAR.picks 만 (smart_money_tracker.py → system_config.json)
estimated_avg_f = float(aft.get_smart_money_avg_price_from_ssot(config, code))
```
- 슈퍼노바 헌터가 종목별 "세력 추정 평단"을 가져와 **스나이퍼 진입가/평단 교차검증**에 사용.
- **데이터 없음 영향:** 평단 추정이 0으로 떨어져 해당 교차검증 로직이 비활성.

### (3) 섀도우 성과 — "스마트머니 버프" 승률 검증
`shadow_performance_tracker.py:238` `evaluate_smart_money_buff()`:
- 가상매매 이력의 `satellite_tags`에 `SMART_MONEY_ACTIVE=yes`(라다 활성 시점 진입, `shadow_tracking.py:250`에서 부착)가 있는 트레이드와
  없는 트레이드의 **청산 승률을 비교**한다.
- 산출: `win_rate_tagged`, `win_rate_untagged`, `delta_pct_pts` → `SHADOW_PERFORMANCE.smart_money_buff`에 저장.
- 리포트 표시(`forward/shared.py:599`):
  `🔥 스마트머니 버프: 태그 종목 승률 X% (일반 Y%, 우위 +Z%p)` / 표본 부족 시 *"표본 부족"*.
- **데이터 없음 영향:** 태그가 안 붙어 `n_tagged_matched=0` → 항상 *"표본 부족"* → 진화 피드백 단절.

### (4) 거시 시너지 배수 — 최종 점수 가중 (tailwind)
`macro_context_snapshot.py`:
- `_shadow_smart_delta()`(:138)가 `smart_money_buff.delta_pct_pts`를 읽어 스냅샷의 `shadow.smart_money_delta_pct_pts`로 노출.
- `compute_clamped_synergy_multiplier()`(:208, :256):
  ```python
  if delta_pct_pts >= 2.0(sh_th):
      mult *= 1.0 + min(sh_bonus=0.02, 0.05)   # shadow_smart_money_tailwind
  mult = clamp(mult, 0.85, 1.08)
  ```
  → 스마트머니 태그가 일반 대비 **+2%p 이상 우위**면 최종 점수에 **약 +2%(상한 +5%) 순풍 배수**.
- **데이터 없음 영향:** delta가 None → tailwind 미적용 → 시너지 배수에서 스마트머니 항이 빠짐.

### (5) 메타 거버너 진화 입력 — META_SATELLITE_INTEL
`meta_governor.py:955`:
```python
sm = cfg.get("SMART_MONEY_RADAR"); picks = sm.get("picks") ...
self._working["META_SATELLITE_INTEL"] = {
    "smart_money_codes": list(picks.keys())[:120],
    "smart_money_n": len(picks), ...
}
```
- 메타 거버너가 위성망(스마트머니·오답노트·센티) 스냅샷을 학습 입력으로 흡수.
- **데이터 없음 영향:** `smart_money_n=0`, `codes=[]` → 진화 튜닝의 위성 입력 품질 저하
  (docs/KR_진화튜닝_구조감사.md:319의 "진화 입력 품질 저하"와 동일 맥락).

### (6) 리포트 위성 브리핑 [0] & 전략 시너지 문구
`satellite_intel_brief.py`:
- `[0/9]` 위성 라인(§2.1) + `collect_satellite_intel_metrics()`(:502)의 `smart_money_count` +
  전략 인사이트(`build_strategy_insight_html`, :587)의 "n종목 매집" 문구.
- `forward/deep_dive.py:135`도 `smart_money_count`를 딥다이브 지표로 사용.

### 기여 요약도
```
smart_money_tracker.py (16:10, KR pykrx)
        │  writes
        ▼
system_config.json: SMART_MONEY_RADAR.picks{ code: {avg_price, divergence_score} }
        │
        ├─(1) try_add_virtual_position ──> ±3% 평단 일치 시 [🕵️세력매집_교차검증] 태그 (전 스캐너 공통 관문)

        ├─(5) meta_governor ─────────────> META_SATELLITE_INTEL (진화 학습 입력)

```

## 4. "데이터 없음"일 때 실제로 무엇이 degrade 되나

| 기능 | 정상 | 데이터 없음 |
|---|---|---|
| 진입 교차검증 배지 (1) | 평단 ±3% 종목에 `[🕵️세력매집_교차검증]` | **한 번도 안 붙음** |
| 스나이퍼 평단 (2) | 세력 평단 기반 교차검증 | `avg_price=0` → 비활성 |
| 스마트머니 버프 승률 (3) | 태그/비태그 승률 비교 | 표본 0 → 항상 *"표본 부족"* |
| 시너지 순풍 (4) | delta≥2%p 시 +2% 가중 | tailwind 항 제거 |
| 진화 입력 (5) | `smart_money_n` > 0 | 0 → 위성 입력 품질↓ |
| 리포트 [0] (6) | "n종목 매집 포착" | "데이터 없음" |

> 정리: 스마트머니 공백은 **진입을 막지는 않지만**(가점성 신호이므로), 신호 신뢰등급·점수 순풍·진화 학습의
> 한 축을 통째로 비활성화한다. 즉 "있으면 더 정교해지는" 보강 레이어가 꺼진 상태로 운용된다.

## 5. 복구 권고

### 5.1 US (스마트머니 데이터 없음)
- **현 상태가 의도된 미구현**이다. "데이터 없음 — US 위성 P2 예정" 문구가 이를 명시한다.
- 구현하려면 US 기관 플로우 소스(예: 13F 분기 보유, ETF 자금 흐름, 다크풀/블록 트레이드 프록시)를 붙여
  `_provider_smart_money_us_placeholder`를 실제 provider로 교체해야 한다. → 별도 과제(P2).

### 5.2 KR (스마트머니 데이터 없음) — 즉시/영구 분리
1. **즉시 채우기 (수동 1회):**
   ```bash
   cd ~/dante_bots/Dual-Screener-Bot
   source venv/bin/activate
   python smart_money_tracker.py
   # → system_config.json 의 SMART_MONEY_RADAR.picks 채워짐. 0건이면 pykrx/네이버 차단 여부 점검.
   ```
2. **데몬 상태 확인 (16:10 잡이 발화하려면 데몬이 살아 있어야 함):**
   ```bash
   systemctl is-active dante-factory     # active 여야 16:10 위성 잡 발화
   journalctl -u dante-factory --since "16:00" | grep -i smart_money
   ```
3. **영구 수정 — ✅ 구현 완료 (옵션 A):** `smart_money_tracker`를 factory 일급 잡으로 이관했다.
   - `factory_pipelines.py`: `smart_money_refresh` 모드 + `_step_smart_money_refresh`(KR `run_smart_money_tracker` + US `run_us_dark_pool_proxy`).
   - `factory.sh --smart-money` CLI + `deploy/generate_factory_crontab.py` **KST 16:10 월–금** 크론 라인(락 직렬화).
   - 함께 이관: `--limit-up`(15:40·16:20 상한가 부검), `--doomsday`(17:00 둠스데이) → 위성 잡 전반이 데몬 생사와 무관하게 cron으로 발화.
   - **0건 방어:** `run_smart_money_tracker`/`run_us_dark_pool_proxy`가 픽 0건이어도
     `{"status":"no_smart_money_today","picks":{}}`로 **명시 초기화** → stale 라다 오인 방지.
   - **US 데이터 주입:** `_provider_smart_money_us_placeholder` 제거, 다크풀 프록시
     (저변동성+거래량 200%↑ 폭증 = 기관 매집)로 `SMART_MONEY_RADAR_US` 적재 후 표시.

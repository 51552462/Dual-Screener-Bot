# 둠스데이 레이더(Macro Doomsday Radar) — 퀀트 구조 진화·발전 기여 검토

> 질문: *"둠스데이 레이더가 지금 우리 퀀트 구조에 발전·진화 부분을 어떻게 기여하는지? 진짜로 기여하는지?"*
>
> 결론 먼저: **진짜로 기여한다.** 단순 알림 봇이 아니라, ① 자본 방어(스캔 차단·신호 게이트), ② 인버스(숏) 모드 자동 전환, ③ **진화 학습 루프로의 데이터 환류**, ④ 관제 브리핑의 4개 채널로 메인 퀀트 엔진에 실제로 배선되어 있다. 특히 ③번이 "진화" 기여의 핵심이다.
>
> 검토 범위(코드 실측): `macro_doomsday_bot.py`, `doomsday_bridge.py`, `factory_pipelines.py`, `system_auto_pilot.py`, `supernova_hunter.py`, `shadow_tracking.py`, `satellite_intel_brief.py`, `evolution_digest.py`, `forward/shared.py`.

## 2. 레이더 엔진 구조 (`macro_doomsday_bot.py`)

**완전 격리 위성(Zero-Coupling).** 메인 스크리너·`market_data.sqlite`를 일절 건드리지 않고, yfinance에서 8개 거시 티커만 독립적으로 받아 `doomsday_status.json` 하나만 산출한다. 메인 엔진이 죽어도 레이더는 죽지 않고, 레이더가 죽어도 메인 엔진은 죽지 않는다(장애 격리).

입력 8티커 → 6개 거시 스트레스 지표 → 롤링 252일 Z-score:

| 축 | 지표 | 의미 |
|---|---|---|
| US | `^TNX − ^IRX` 금리 스프레드 + 20일 모멘텀 | 장단기 역전/신용 사이클 |
| US | `HYG/IEF` (하이일드/국채) | 신용 스프레드(위험선호) |
| US | `HG=F / GC=F` (구리/금) | 실물 경기 vs 안전자산 |
| KR | `KRW=X` 10일 ROC | 원화 약세(자본 이탈) |
| KR | `^KS11` 20일 ROC | 코스피 추세 |

```119:160:macro_doomsday_bot.py
def run_macro_doomsday_radar() -> dict:
    ...
    us_doom = _doom_scalar_from_z(lz_us)
    kr_doom = _doom_scalar_from_z(lz_kr)
    global_score = float(np.clip(0.6 * us_doom + 0.4 * kr_doom, 0.0, 100.0))
    regime = _classify_regime(global_score=global_score, kr_score=kr_doom)
```

- `Doom Score = mean(|Z|) × 32`, 0~100 클립. 거시 변수들이 평소 분포에서 얼마나 꼬리(tail)로 벗어났는지를 점수화.
- `Global_Contagion = 0.6·US + 0.4·KR` (전염 가중).
- 레짐: `DOOMSDAY`(Global>70) / `DEFENSIVE_KR`(41≤KR≤70) / `ELEVATED` / `BULL`(Global≤40).

## 3. 데이터 파이프라인 (위성 → SSOT)

```
[cron 17:00 KST 월–금]            (factory.kr.crontab.example:37)
   └─ factory.sh --doomsday → MODE=doomsday_radar
        └─ factory_pipelines._step_doomsday_radar → macro_doomsday_bot.main()
             ├─ doomsday_status.json 원자적 저장
             └─ doomsday_bridge.ingest_doomsday_status_file()
                   └─ sync_doomsday_to_system_config()
                         ├─ system_config["DOOMSDAY_DEFCON"]   ← DEFCON 1~5 블록
                         ├─ system_config["DOOMSDAY_RADAR_SSOT"]
                         └─ INVERSE_MODE 동기화 (+ 옵션 인버스 사이클)
```

추가로 **매 스캔 사이클 꼬리**에서도 가벼운 동기화가 돈다(`factory_pipelines.py:869 tail_doomsday`, `_step_doomsday_bridge_sync`). 즉 17:00 본스캔뿐 아니라 장중 스캔마다 최신 DEFCON이 config에 반영된다.

**중요 — 데몬 탈피 이관 완료:** 과거 17:00 둠스데이 잡은 불안정한 `--daemon` 루프에 갇혀 있었으나, 현재는 `factory.sh --doomsday` 정식 크론 잡(락 직렬화)으로 이관되어 시스템 스캔과 충돌 없이 실행된다(`generate_factory_crontab.py:68`).

### DEFCON 매핑 (`doomsday_bridge.defcon_level_from_payload`)
| DEFCON | 조건 | 효과 |
|---|---|---|
| 1 (최악) | Global ≥ 82 | **스캔 전면 중단** |
| 2 | DOOMSDAY / Global ≥ 70 | 신호 산출 유지+가상매매 게이트 soft-fail, 인버스 ON |
| 3 | DEFENSIVE_KR (41≤KR≤70, Global>40) | 인버스 동기화 진입 |
| 4 | 그 외 | 평시 |
| 5 (안전) | BULL / Global ≤ 40 | 공격 허용 |

## 4. 실제 소비 채널 (= "진짜 기여"의 증거)

### A. 자본 방어 — 스캔 차단 + 신호 게이트
```1536:1558:supernova_hunter.py
        _dd = config.get('DOOMSDAY_DEFCON') or {}
        defcon = _dd.get('level', 5) if isinstance(_dd, dict) else 5
        ...
        # DEFCON 1만 스캔 전면 중단. 2 이하는 신호 산출 유지(가상매매 gate에서 soft-fail).
        if defcon <= 1:
            ...
            funnel.drop("DOOMSDAY_HALT")
            return {...}
```
DEFCON 1이면 **모든 종목 워커가 신규 진입을 거부**한다. 이는 폭락 초입에서 신규 매수를 0으로 만들어 Live NAV의 급락을 차단 — 복리 자산곡선의 하방을 직접 방어한다.

### B. 인버스(숏) 모드 자동 전환
```260:277:system_auto_pilot.py
    dd = current_config.get("DOOMSDAY_DEFCON") or {}
    ...
    defcon_crash = lvl <= 2
    ...
    current_config["INVERSE_MODE_ACTIVE"] = bool(defcon_crash or meta_crash or vix_crash)
```
DEFCON≤2면 `INVERSE_MODE_ACTIVE=True`로 전환되고, 브리지가 인버스 ETF 스나이퍼 1사이클을 돌린다(`doomsday_bridge.py:147`). 단순 "쉬기"가 아니라 **하락장에서 돈을 버는 대칭 알파**를 가동한다.

### C. 진화 학습 환류 (← 사용자가 물은 "진화" 기여의 핵심)

둠스데이 신호는 3가지 경로로 **학습 데이터**가 되어 시스템을 진화시킨다:

1. **차단거래 반사실 학습**
   ```1553:1555:supernova_hunter.py
   shadow_tracking.record_blocked_trade(
       code, _halt_name, "DOOMSDAY_DEFCON", 0.0
   )
   ```
   DEFCON 때문에 막은 거래를 `virtual_trade_history`에 "막지 않았다면?"으로 기록한다. `shadow_performance_tracker`가 이를 추적·평가하여 **"둠스데이 차단이 실제로 손실을 막았는가"**를 사후 채점한다. → 방어 임계치의 타당성을 데이터로 검증.

2. **위성태그 각인 (모든 실거래에 DEFCON 도장)**
   ```235:245:shadow_tracking.py
   def build_satellite_tags(config: dict) -> str:
       ...
       parts.append(f"DEFCON={int(dd.get('level', 5))}")
   ```
   `supernova_hunter.py:2037`에서 모든 진입 거래에 `satellite_tags`(DEFCON=N 포함)가 각인된다(`forward/shared.py:2229`). 이 태그는 주간 Flow 태그 딥다이브·독성 페널티 학습의 원천이 되어, **"어떤 거시 국면(DEFCON)에서 잡은 거래가 독성이었나"**를 베이지안으로 학습한다(앞서 구축한 유동적 독성 페널티 엔진과 직결).

3. **진화 다이제스트 Δ 추적**
   ```31:37:evolution_digest.py
   keys = (
       "DYNAMIC_SUPERNOVA_CUTOFF", "DYNAMIC_ML_BOX_CUTOFF",
       "INVERSE_MODE_ACTIVE", "DOOMSDAY_DEFCON", "META_GLOBAL_KELLY_MULT",
   )
   ```
   `DOOMSDAY_DEFCON`·`INVERSE_MODE_ACTIVE` 변화가 `[Δ] 진화·튜닝` 다이제스트에 자동 기록되어, 시스템 상태 전이의 감사 추적(audit trail)을 남긴다.

### D. 관제·리포트 브리핑
```655:664:satellite_intel_brief.py
    if defcon <= 2:
        strategy_insight += "🚨 [폭풍 전야] ... 신규 매수 전면 중단, 현금 비중 극대화 ...\n"
    elif defcon >= 4 and sm >= 5:
        strategy_insight += "🚀 [골디락스 공격] ... 공격적 롱 베팅 권장 ...\n"
```
DEFCON과 스마트머니 수급을 **결합**해 공격/방어 전략을 자동 서술한다. 일일 `[0]` 배너에도 `DEFCON n/5`가 노출된다.

## 5. "진화"에 기여하는 정량 경로 요약

```
거시 꼬리위험 감지(레이더)
   → DEFCON↓ → 신규 진입 차단 + 인버스 ON
        → Live NAV 하방 방어 (복리 분모 보존)
             → 다음 주 가상 자본곡선 E_t 상향  [Live NAV/Compounding 엔진]
   → 차단·태그 데이터가 shadow/flow-tag 학습으로 환류
        → 베이지안 독성 페널티·캐리 필터가 "위험 국면 패턴"을 학습
             → 다음 사이클의 켈리·태그 가중치 자가 갱신  [진화]
```
즉 둠스데이 레이더는 **수익(알파) 생성기가 아니라 "생존 + 학습 피드"** 역할로 진화에 기여한다. 헤지펀드 관점에서 진화의 1순위 전제는 *살아남아 복리를 이어가는 것*이며, 레이더는 그 분모(자본)와 학습 표본(태그)을 동시에 지킨다.

## 6. 한계 및 점검 포인트 (정직한 평가)

- **외부 데이터 의존:** yfinance 8티커 중 4개 미만 수집 시 `RuntimeError`로 패널 구성 실패 → `regime="ERROR"`. 이 경우 기존 config DEFCON이 유지된다(스테일). `errors[]` 필드로 추적.
- **켈리 직접 변조는 없음:** DEFCON이 `META_GLOBAL_KELLY_MULT`를 직접 곱하지는 않는다. 방어는 "스캔 차단 / soft-fail / 인버스"라는 이산적(binary-ish) 게이트로 작동한다. (켈리 연속 감쇠까지 원하면 향후 확장 여지.)
- **격리의 대가:** 레이더는 메인 DB를 안 보므로 개별 종목·포지션 맥락은 모른다. 순수 톱다운 거시 신호만 제공한다(설계 의도).

### 점검 명령어 (서버)
```bash
# 1) 레이더 최신 산출물 확인

# 3) 수동 1회 강제 실행(데몬 무관)

python3 -c "import sqlite3,os; from market_db_paths import report_db_read_path as P; c=sqlite3.connect(P()); print(c.execute(\"SELECT sig_type,COUNT(*) FROM virtual_trade_history WHERE sig_type LIKE '%DOOMSDAY%' GROUP BY sig_type\").fetchall())"
```

## 7. 결론

- **진짜 기여하는가? → 그렇다.** 코드상 4개 채널 모두 실제로 메인 엔진에 배선되어 있으며 죽은 코드가 아니다.
- **진화 기여의 본질:** 알파를 직접 만들기보다 *(1) 폭락기 자본 방어로 복리 곡선의 하방을 지키고, (2) 차단/태그 데이터를 학습 루프(shadow·flow-tag·evolution digest)로 환류*시켜 시스템이 "위험 국면"을 스스로 학습하게 한다.
- **데몬 이관 완료**로 과거의 "실행 안 됨" 리스크는 해소되었고, 매 스캔 꼬리 + 17:00 정시 크론의 이중 동기화로 신선도가 보장된다.
- 보강 여지: DEFCON→켈리 연속 감쇠 연동, 레이더 ERROR 시 스테일 경보 강화.

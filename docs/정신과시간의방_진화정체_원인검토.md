# 정신과 시간의 방(Hyperbolic Time Chamber) 운영·진화 정체 원인 검토

> **범위:** KR·US 공용 R&D 챔버 (`synthetic_data_generator.py` → `incubator_engine.py` → `mutant_oos_validator.py` → `mutant_pending_bridge.py`). **bitget 폴더 제외.**
> **방법:** 코드 정독 + **로컬 실제 1회 재현 실행**(2026-06-28). 본 문서는 검토·증거 기록용이며, 코드는 변경하지 않았다.
> **핵심 결론:** “승률은 좋은데 수익률이 -0.000몇” 은 **버그가 아니라 챔버 설계의 필연**이다. 그리고 그 위에 **진화 파이프라인을 끊어버리는 치명 버그 1건**이 더 있어, 지금 구조는 *돌아가는 것처럼 보이지만 한 발짝도 진화하지 못한다.*

## 0. 한 줄 요약

| 질문 | 답 |
|------|-----|
| 정신과 시간의 방이 제대로 운영되나? | **부분적으로만.** 매주 토요일 배치는 기동되지만, **출력은 매번 똑같은 음(-) 수익 5종**이고, **OOS 게이트에서 100% 크래시**한다. |

## 1. 정신과 시간의 방의 구조 (무엇이 무엇인지)

“정신과 시간의 방”은 **실전 장부/스캐너와 완전히 격리된** 합성 R&D 챔버다. KR/US 시장 구분이 없는 **공용 모듈**이다.

```
[토 00:00]  synthetic_data_generator.py
            └─ Merton 점프-확산 합성 일봉 100종목 × 1000일 생성
               → synthetic_market.sqlite (synthetic_ohlcv)

[토 02:00]  incubator_engine.py   ← "🏆 [정신과 시간의 방]"
            └─ 합성 큐브(1000×100)에 전략 expr 벡터 평가
               → mutant_hall_of_fame.json  (win_rate/avg_return 랭킹)

[토 03:00]  mutant_oos_validator.py
            └─ hall_of_fame 상위 5종을 실데이터(market_data.sqlite, RO)로 재검증
               → validated_live_mutants.json (합격분만 promoted[])

[토 03:10]  mutant_pending_bridge.py
            └─ promoted[] → system_config["PENDING_MUTANTS"] (대기열)
               → (수동 승인 게이트) APPROVE_PENDING_MUTANTS_TO_INCUBATOR=true 일 때만
                 → INCUBATOR_TEMPLATES 병합

[매일/매주] system_auto_pilot.py 엔진 14
            └─ INCUBATOR_TEMPLATES 를 실전 forward_trades 로 심판 → 정규직(NEW_EVOLUTION) 승격
```

스케줄러 위치: `system_auto_pilot.py` L1946~1991 (토요일 00:00 / 02:00 / 03:00 / 03:10).

## 2. 재현 증거 (로컬 1회 실행, 2026-06-28)

`synthetic_data_generator.py` → `incubator_engine.py` 를 그대로 1회 돌린 **실제 산출물**:

| 순위 | 전략 | 시그널 수 | 승률 | 익일 평균수익률 |
|------|------|-----------|------|-----------------|
| 1 | M3_bull_range (`close > low*1.02 & close > open`) | 24,097 | **50.06%** | **-0.000186** |
| 2 | M5_vol_breakout | 2,738 | 50.00% | -0.000278 |
| 3 | M4_upper_wick_pressure | 43,242 | 49.66% | -0.000453 |
| 4 | M2_weak_close_high_vol | 16,788 | 49.51% | -0.000596 |
| 5 | M1_gap_up | 688 | 47.53% | -0.001808 |

→ **모든 전략이 승률 ~50%, 평균수익 음수.** 사용자가 본 “승률 좋은데 -0.000몇”이 그대로 재현됨.

그리고 **합성 시장 자체의 무조건(랜덤) 익일 수익률**을 직접 측정하면:

```
baseline next-day mean ret = -0.000397   ← 시장 전체의 공짜 기대수익이 이미 음수
baseline median ret        = -0.000150
baseline win-rate (>0)     = 49.68%
mean < median  →  음의 왜도(negative skew) = True
```

**즉, 전략들이 내는 -0.0004 안팎은 “전략의 실력”이 아니라 “이 합성 세계의 기본 중력”이다.** 전략은 알파를 못 만들고 시장 평균을 그대로 베끼고 있을 뿐이다.

## 3. 왜 “승률 좋은데 수익률 -0.000몇”인가 (근본 원인)

### 원인 A — 합성 시장이 애초에 ‘지는 카지노’다 (가장 큰 원인)

`synthetic_data_generator.py` L66~70 파라미터:

```python
mu        = 0.00015   # 일간 추세 (거의 0)
sigma     = 0.018
lam       = 0.035     # 점프 발생률
mu_jump   = -0.02     # 점프는 '평균적으로 하락' 쪽으로 편향
sigma_jump= 0.045
```

로그수익 기대값 ≈ `mu + lam·mu_jump = 0.00015 + 0.035×(-0.02) = -0.00055` → 단순수익 환산 약 **-0.0004/일**.
- **점프가 음(-)으로 편향**(`mu_jump=-0.02`)되어 있어, 드물지만 큰 하락이 평균을 끌어내린다.
- 그 결과 **중앙값 > 평균** (음의 왜도): 대부분의 날은 작게 오르니 **승률은 ~50%**, 하지만 가끔의 큰 음봉이 **평균을 음수**로 만든다.

> 이건 “고승률·저(음)기대” 전략의 교과서적 패턴이다. 챔버가 **그런 세계만 만들어 주고 있어서**, 거기서 뽑히는 챔피언도 자동으로 “자주 이기지만 길게 보면 잃는” 전략이 된다.

### 원인 B — 랭킹이 승률을 1순위로 둔다

`incubator_engine.py` L292~295:

```python
ranked = perf.sort_values(
    by=["win_rate", "avg_return", "n_signals"],
    ascending=[False, False, False],
)
```

**승률(win_rate)이 1차 정렬 키**이고 수익률은 2차다. 그래서 음의 기대수익이라도 **승률이 높은 쪽이 명예의 전당 1등**이 된다. 위 표에서 1등 M3(-0.000186)은 “덜 잃는” 전략일 뿐 “버는” 전략이 아니다. **수익성이 아니라 ‘덜 지는 능력’으로 챔피언을 뽑고 있다.**

### 원인 C — ‘돌연변이’가 사실상 없다 (정적 5종 반복)

`run_incubator()`는 `incubator_engine.main()`에서 **인자 없이** 호출된다 → 항상 `default_mock_strategies()`의 **하드코딩 5종(M1~M5)** 만 평가한다 (L271~279, L287).

- 유전 연산(교배/변이/무작위 expr 생성)이 **전혀 없다.**
- 매주 합성 데이터는 새로 만들지만(시드 고정이라 통계분포는 동일), **전략 풀은 영원히 M1~M5.**
- 따라서 **매주 같은 5종을 같은 음(-)세계에서 다시 줄세우는 것**이 전부다. 이건 정의상 진화가 아니라 **반복**이다.

> 참고: `forward/shared.py`의 `generate_mutant_strategies()`(MUTANT_1~3, 가우시안 유전자 변이)는 **이름만 ‘인큐베이터’일 뿐 다른 시스템**이다. 그것은 expr 문자열이 아니라 **파라미터(cpv/tb/bbe/rs)** 를 변이시켜 `INCUBATOR_TEMPLATES`에 직접 쓴다. 즉 “정신과 시간의 방(expr 챔버)”과 “파라미터 인큐베이터”는 `INCUBATOR_TEMPLATES`라는 키만 공유할 뿐 **혈통이 다르다** (§5).

## 4. 진화가 ‘발전’으로 이어지지 못하는 치명 단절

설령 위 A~C를 고쳐 좋은 챔피언이 나와도, **다음 관문에서 100% 막힌다.**

### 단절 1 — OOS 검증기가 항상 `NameError`로 죽는다 ⛔ (치명)

`mutant_oos_validator.py` L361:

```python
"data_source": "market_data.sqlite_ro" if os.path.exists(MARKET_DB) else "fdr_fallback",
```

여기서 **`MARKET_DB`는 이 파일에 정의된 적이 없다.** (import는 `from market_db_paths import market_db_read_path` 뿐 — `MARKET_DB`라는 이름은 어디에도 없음.)

→ `run_oos_validation()`이 패널 로딩·평가를 다 끝낸 **마지막 payload 조립 줄에서 매번 `NameError`** 발생
→ `main()`의 `except`가 받아서 **`validated_live_mutants.json`에 `{"promoted": []}` + error**만 기록
→ `mutant_pending_bridge`는 “promoted 비어 있음”으로 종료
→ **`PENDING_MUTANTS`에 신규 전략이 단 한 건도 들어가지 못한다.**

**결과: 매주 토 03:00 OOS 단계에서 파이프라인이 끊긴다. 챔버 결과가 실전 후보로 넘어가는 길이 완전히 막혀 있다.** 이것이 “진화가 안 된다”는 체감의 가장 직접적인 코드 원인이다.

> 고치는 법은 단순: 상단 상수로 `MARKET_DB = market_db_read_path()` 를 정의하거나, L361을 `os.path.exists(market_db_read_path())`로 바꾸면 됨. (본 검토에서는 코드 미수정.)

### 단절 2 — 실전 병합은 기본 OFF (수동 승인 게이트)

OOS를 통과해 `PENDING_MUTANTS`까지 갔다 해도, `INCUBATOR_TEMPLATES`로의 실제 병합은 `APPROVE_PENDING_MUTANTS_TO_INCUBATOR == True` 일 때만 일어난다 (`mutant_pending_bridge.apply_pending_mutants_if_approved`). 이 플래그는 **기본 OFF**라 운영자가 켜지 않으면 영원히 대기열에만 쌓인다. (이는 안전장치로서 의도된 설계지만, 단절 1과 겹치면 “대기열조차 비어 있는” 이중 정체가 된다.)

### 단절 3 — OOS 임계치 vs 음(-)세계 챔피언의 모순

OOS 합격 기준(L32~33): `win_rate > 0.60` **그리고** `avg_return > 0.0`.
그런데 §3에서 보듯 챔버가 올려보내는 챔피언은 **승률 ~50% / 수익 음수**. 설령 단절 1을 고쳐도 **이 챔피언들은 OOS 게이트를 통과할 수 없다.** (실데이터는 추세가 +라 일부 통과 가능성은 있으나, 입구에서 “덜 지는 전략”을 뽑는 한 합격률은 구조적으로 낮다.)

## 5. 용어 혼선 정리 — ‘인큐베이터’가 두 개다

“진화가 되는 줄 알았는데 안 된다”는 혼동의 절반은 **같은 이름 두 시스템** 때문이다.

| 구분 | (A) 정신과 시간의 방 (expr 챔버) | (B) 파라미터 인큐베이터 |
|------|-------------------------------|------------------------|
| 코드 | `incubator_engine.py` 등 4종 | `forward/shared.generate_mutant_strategies()` |
| 변이 대상 | **수식 문자열**(`close>open*1.05`) | **유전자 파라미터**(cpv/tb/bbe/rs) |
| 데이터 | 합성 시장 (synthetic_market.sqlite) | 실전 부모 S급 DNA |
| 산출 | mutant_hall_of_fame → validated → PENDING | `INCUBATOR_TEMPLATES`에 직접 기입 |
| 실전 심판 | (수동 승인 후) 엔진 14 | 엔진 14 (`system_auto_pilot` L1696~1795) |
| 현재 상태 | **단절 1로 막힘** | 동작하지만 표본/PF 게이트 의존 |

엔진 14가 실제로 “정규직 승격”을 집행하는 대상은 **`INCUBATOR_TEMPLATES`** 인데, 현재 그 키를 채우는 건 거의 (B)뿐이다. (A)의 기여는 단절 1·2 때문에 **0**이다. 즉 **“정신과 시간의 방”은 실전 진화에 사실상 기여하지 않고 있다.**

## 7. “제대로 운영/진화 중인가?” 최종 판정

| 항목 | 판정 | 근거 |
|------|------|------|
| 배치 기동 | 🟡 **돌긴 함** | 토 00/02/03시 스케줄 존재, 재현 시 생성·평가 정상 |
| 전략 다양성(진화) | 🔴 **없음** | 고정 5 mock 반복, 돌연변이 생성 미연결 (원인 C) |
| 수익성 신호 | 🔴 **음(-)** | 합성 시장 자체가 -0.0004 드리프트 (원인 A) |
| 챔피언 선발 품질 | 🔴 **왜곡** | 승률 1순위 랭킹이 ‘덜 지는’ 전략 선발 (원인 B) |
| 실전 연결(발전) | 🔴 **단절** | OOS `NameError`로 승격 0건 (단절 1) + 수동 게이트 OFF (단절 2) |

**종합: “정신과 시간의 방”은 시계는 돌지만 수련생이 한 명도 졸업하지 못하는 방이다.** 표면적으로 매주 리포트(“🏆 인큐베이터 평가 완료”)는 오지만, 그 내용은 **항상 같은 음(-)수익 5종**이고 실전으로 단 한 건도 넘어가지 못한다. 따라서 **진화·발전은 사실상 0.**

KR/US 모두 동일하다 — 이 챔버는 시장 무관 공용 모듈이라 **양쪽 다 똑같이 정체**한다. (KR/US 각자의 실전 진화 루프 자체는 `INCUBATOR_TEMPLATES`/엔진 14/MetaGovernor로 별개로 살아 있으나, **챔버로부터의 신규 알파 공급은 끊겨 있다.**)

## 8. 권고 (우선순위 — 코드 변경은 별도 승인 후)

1. **[P0·즉시] OOS `NameError` 수정** — `mutant_oos_validator.py` 상단에 `MARKET_DB = market_db_read_path()` 정의(또는 L361을 함수 호출로 교체). 이거 하나만 고쳐도 “승격 0건” 단절이 풀린다. **가장 비용 대비 효과 큼.**
2. **[P0] 합성 시장 드리프트 교정** — `mu_jump=-0.02`(음 편향)를 0 부근으로, `mu`를 소폭 +로. 목표: **무조건 기대수익 ≈ 0**이 되도록 보정해야 “전략의 알파”와 “시장 중력”이 분리된다. (지금은 시장 중력이 -0.0004라 모든 전략이 음수로 보임.)
3. **[P0] 랭킹 기준을 기대값 중심으로** — `sort_values`를 `avg_return`(또는 Sharpe/기대값×시그널수) **1순위**로. 승률은 보조 지표로. “자주 이기는” 대신 “길게 버는” 전략을 뽑게.
4. **[P1] 진짜 돌연변이 생성기 연결** — expr 문자열을 무작위 조합/교배/변이로 생성해 `run_incubator(strategies=...)`에 주입. 고정 5종 반복을 끝내야 ‘진화’가 성립.
5. **[P1] OOS 임계치 현실화** — `avg_return>0` 단독 게이트는 음세계 챔피언과 모순. 시장 드리프트 보정(권고 2) 후 **시장 초과수익(alpha) 기준**으로 재설계.
6. **[P2] 합성 데이터 다양성** — 매주 다른 시드/국면(상승·하락·횡보·고변동)을 섞어 과적합 방지.
7. **[P2] 두 인큐베이터 명칭 분리** — (A) expr 챔버와 (B) 파라미터 인큐베이터를 문서·로그·텔레그램에서 다른 이름으로 표기해 운영 혼선 제거.

## 12. 🚀 전면 재구축 완료 로그 (기관급 자율 진화 챔버)

§8 권고가 **모두 코드로 구현·로컬 검증**되었다. 아래는 미션별 결과.

### Mission 1 — 긴급 버그 수정 & 랭킹 정상화
- `mutant_oos_validator.py`: `MARKET_DB = market_db_read_path()` 정의로 **361행 NameError 제거**(실데이터 read-only 검증으로 검증됨, `data_source=market_data.sqlite_ro`).
- `synthetic_data_generator.py`: 정상분포 가중 무조건 드리프트를 자동 보정 → **baseline ≈ +0.00004**(기존 -0.0004 에서 0 수렴). 공정 테스트 베드 확보.
- `incubator_engine.py`: 랭킹을 **샤프(Sharpe) 1순위 · 기대값(Expectancy) 2순위**로 교체. 이제 '덜 지는' 전략이 아니라 **초과 알파(excess>0) 전략**이 명예의 전당에 오른다.

### Mission 2·3 — AST 유전 알고리즘 + 동적 변속 기어
- 신규 `genetic_expr_builder.py`: `ast` 트리 단위 **생성/교배(crossover)/돌연변이(mutation)**. 고정 5종 mock 폐기, 매주 **N(기본 1,000)개 돌연변이 군집** 창조. 모든 산출물은 `is_safe_expression` 게이트 통과.
- `regime_gear()`: BULL/SIDEWAYS=교배 0.8(수렴), HIGH_VOL/BEAR=돌연변이/신규 상향(탐색), **챔피언 생존율 낮을수록 탐색 강제 가산**. 직전 세대 챔피언을 부모로 상속(엘리트 보존).

### Mission 4 — 다중 국면 합성 생태계(HMM)
- Bull/Bear/Sideways/Black Swan 4국면 + 전이행렬 → 정상분포 `{SIDE .54, BULL .28, BEAR .17, SWAN .006}`. 일자별 국면 경로로 점프-확산 모수 전환. `synthetic_meta` 에 국면 혼합·보정값 기록.

### Mission 5 — 자율 승격 + 밴딧 탐색 밸브
- OOS 게이트를 **실데이터 베이스라인 대비 초과 알파**로 지능화(`excess > min_excess_alpha & wr>0.5 & n≥30`). 로컬 실데이터에서 음(-)초과 전략은 정확히 탈락, 양(+)초과만 승격됨을 확인.
- `mutant_pending_bridge.auto_merge_validated_into_incubator()`: **수동 게이트 우회** → 합격 유전자를 `INCUBATOR_TEMPLATES`(`GP_MUT_*`, `capital_mode=EXPLORE_MIN_KELLY`)에 즉시 병합 + `template_bandit.init_exploration_arm()` 으로 **최소 켈리(0.1) 탐색 밸브**만 개방. 가치 증명 시 베이지안으로 자본 확대.

### Mission 6 — 용량 최적화형 군집 면역 백신
- 신규 `clustered_immune_vaccine.py`: 실패 템플릿을 toxic bbox 로 등록하되 **용량 상한(기본 500)**. 초과 시 **K-Means(폴백 Agglomerative)** 군집화 → 각 군집 **spanning bbox 백신(centroid)** 만 남기고 원본 영구 삭제. 로컬 검증: **700개 → 64 백신(O(K))**. 오토파일럿 **토 03:20 Priority 3** 배치 연결.

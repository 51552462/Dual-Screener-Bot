# 초신성(Supernova) — 수익 기여 여부 & 미래 초신성과의 자가진화 검토

> 질문: *"초신성 Top 1·2·3가 실제 수익률에 기여하는가? 미래 초신성들의 수치와 얼마나 부합하며, 부합하지 않고 수익이 안 나올 때 실제 미래 초신성과 비슷해지게끔 스스로 진화·발전하는 헤지펀드·기관급 로직이 들어있는가?"*
>
> 검토 범위(코드 실측): `supernova_hunter.py`, `system_auto_pilot.py`(엔진 6·9·10), `elastic_threshold.py`, `forward/shared.py`(try_add_virtual_position), `forward/ledger.py`, `limit_up_forensics.py` → `forensics_pioneer.py`, `forward/dna_autopsy.py`([6/9]).
>
> **결론 먼저:**
> 1. **기여한다 (✅).** Top 픽은 "초신성 DNA 일치율(%)"로 점수화되어 컷오프 통과 시 `forward_trades`에 `sim_kelly_invest`(켈리 노출액)와 함께 편입 → 실제 PnL·Live NAV에 직접 반영된다.
> 2. **"부합도"는 곧 일치율(%)이다.** Top 1·2·3의 점수는 그 종목이 과거/실전 초신성들의 4D DNA와 코사인 유사도로 얼마나 닮았는지를 수치화한 값이다.
> 3. **스스로 진화한다 (✅, 4중 루프).** 수익이 안 나면 ① 허들을 높여 더 닮은 놈만 받고(엔진9), ② 안 통하는 템플릿은 도태·국고환수·면역화(엔진10)하며, ③ **실전 고MFE 승자 DNA 쪽으로 템플릿 자체를 α=0.3 스무딩 수렴(엔진6, 핵심)**시키고, ④ 실제 상한가(미래 초신성) DNA를 역추적해 섀도우 검증(forensics)한다.

---

## 1. 초신성 파이프라인 전체 흐름

```
[학습] 과거/실전 TOP 수익 종목 → 4D DNA 추출(extract_dna_from_df)
         → DNA_SUPERNOVA_{KR/US}_MULTI 템플릿 + MFE_진화형_황금타점
[스캔] 오늘 종목 vs 템플릿 코사인 유사도 → final_score(=일치율 %)
         → DYNAMIC_SUPERNOVA_CUTOFF 게이트 통과분만 채택
[기여] try_add_virtual_position → forward_trades (sim_kelly_invest, score)
         → MFE/MAE 청산(ledger) → final_ret → PnL → Live NAV 복리
[진화] 청산 결과(WR/PF/MFE)를 주말·일일 자율조율이 회수 →
         허들·템플릿·황금타점을 실전 초신성 쪽으로 재조정
```

DNA는 3축 벡터다(코드 `ideal_templates`): **CPV**(거래대금/매집 강도), **TB**(추세 바디), **BBE**(볼린저 밴드 확장). 초신성 = 이 DNA가 폭발 직전 종목과 닮은 종목.

## 2. Top 1·2·3는 어떻게 나오고 무엇을 의미하나

`supernova_hunter.execute_supernova_live_scan`에서 각 후보는 여러 템플릿(RANK_A~D, MFE 황금타점 등)과 코사인 유사도를 재고, 최고 일치 템플릿의 점수를 `final_score`로 잡는다:

```1898:1914:supernova_hunter.py
final_score = ml_score * 100 + _scan_bonus_pts
...
final_score = _cos_sim * 100 + _scan_bonus_pts
```

- `final_score`(일치율 %) 상위 = "오늘 가장 초신성 DNA에 가까운 종목" = Top 1·2·3.
- 즉 **Top 순위 = 미래 초신성과의 부합도 순위**다. 별도의 미래 예측치가 아니라, "실제 폭발했던 초신성들의 DNA"에 얼마나 부합하는지를 정렬한 것.

## 4. ★ 자가진화 4중 루프 (헤지펀드·기관급 로직 핵심)

### Loop 1 — 컷오프 자율 튜닝 (엔진 9): "안 닮은 놈을 더 걸러낸다"
승률 기반으로 일치율 허들을 동적 조정한다:

```1289:1298:system_auto_pilot.py
if wr < 0.45: # 승률 낮으면 허들 조이기
    new_val = min(0.90, curr_val + 0.05)
    ... "🚨 [방어력 강화] 허들 상향"
elif wr > 0.65 and len(sub_df) < 10: # 승률 좋은데 표본 적으면 그물 넓히기
    new_val = max(0.40, curr_val - 0.03)
    ... "🔥 [공격적 포착] 허들 하향"
```

- **수익이 안 나면(WR<45%) `DYNAMIC_SUPERNOVA_CUTOFF`를 +5%p 올린다** → 더 높은 DNA 부합도를 요구 = 미래 초신성과 더 닮은 종목만 통과. 이것이 질문의 "부합 안 하면 비슷해지게 진화"의 1차 메커니즘.
- 표본 기아 시 `elastic_threshold.relief_adjust_autonomous_cutoff`로 죽음의 나선(Death-Spiral)을 방지(과도하게 조여 표본이 0이 되는 것 방지).

### Loop 2 — 템플릿 도태 토너먼트 (엔진 10): "안 통하는 초신성 유전자를 죽인다"
```1350:1374:system_auto_pilot.py
if t_wr < 0.35 or t_pf < 1.0:
    # 도태 유전자는 삭제하지 않고 냉동 보관소로 이동
    market_archive[template_name] = {... "wr":..., "pf":...}
    # 안티 패턴 면역 체계: 도태(승률 35% 미만) 템플릿 DNA 축적
    anti_patterns[anti_key] = {"source":"CULLED_TEMPLATE", "cpv":..., "tb":..., "bbe":...}
```
- 승률 35% 미만 또는 PF<1 템플릿은 **냉동보관(ARCHIVED) + 국고 환수**(`:1377` 잔고 역산 → `CENTRAL_TREASURY` 가산)되고, **안티패턴 면역체계(ANTI_PATTERNS)**에 등록되어 향후 비슷한 DNA를 차단한다. = 백신.

### Loop 3 — MFE 황금타점 오토스무딩 (엔진 6): "실전 초신성 DNA로 수렴" ★핵심
실제로 큰 MFE(최대 유리 변동)를 낸 승자들의 실전 DNA를 회수해, 템플릿을 그쪽으로 **점진 수렴**시킨다:

```1238:1253:system_auto_pilot.py
old_mfe_template = current_config.get("DNA_SUPERNOVA_MFE_WEIGHTED", {"cpv": real_cpv, "tb": real_tb})
# (기존 값 * 0.7) + (새로운 실전 값 * 0.3) = 점진적 오토 추적
smoothed_cpv = old["cpv"]*(1-0.3) + real_cpv*0.3
smoothed_tb  = old["tb"] *(1-0.3) + real_tb *0.3
smoothed_bbe = old["bbe"]*(1-0.3) + real_bbe*0.3
current_config["DNA_SUPERNOVA_MFE_WEIGHTED"] = {cpv, tb, bbe, last_updated}
```

수학적으로 지수이동평균(EMA, α=0.3):

\[ T_{t+1} = (1-\alpha)\,T_t + \alpha\, D^{real}_t,\quad \alpha=0.3 \]

여기서 \(D^{real}\)는 **방금 큰 수익을 낸 실전 초신성들의 평균 DNA**. 이 수렴된 템플릿이 스캐너에서 `MFE_진화형_황금타점`으로 즉시 소비된다:

```1375:1377:supernova_hunter.py
mfe_weighted = config.get("DNA_SUPERNOVA_MFE_WEIGHTED")
if mfe_weighted:
    ideal_templates['MFE_진화형_황금타점'] = np.array([mfe_weighted['cpv'], mfe_weighted['tb'], mfe_weighted.get('bbe', 20.0)])
```

→ **이것이 질문의 정답.** 하드코딩 수치를 한 번에 갈아엎지 않고(휩쏘 방지), 실전에서 진짜 폭발한 종목들의 DNA 쪽으로 매주 30%씩 부드럽게 끌려간다. 시간이 지날수록 템플릿이 "미래에 실제로 초신성이 되는 종목"의 모습에 수렴한다. 버전 누적(`DNA_SUPERNOVA_MFE_MULTI`, 최대 10개)으로 세대 진화 이력도 보존.

### Loop 4 — 상한가 역추적 → 미래 초신성 DNA 섀도우 검증 (forensics)
```limit_up_forensics.py → forensics_pioneer.py```
- `limit_up_forensics`: **실제 상한가·급등 종목**(= 그날의 진짜 초신성)을 ~30거래일 역추적해 "선취매 DNA"를 뽑아 `LIMIT_UP_DNA`(상한가 코호트 DNA)로 저장.
- `forensics_pioneer.run_forensics_pioneer`: 오늘 종목 중 그 DNA와 합치하는 것을 `virtual_trade_history`(섀도우)에 기록 → **"우리 초신성 픽이 실제 미래 급등주 DNA를 얼마나 선제 포착했나"를 섀도우로 검증**. 본 장부와 분리된 무위험 학습 트랙.

### 보조 루프
- **타임머신 백테스트 생존 템플릿**(`_load_time_machine_cache` → `surviving_templates`, `supernova_hunter.py:1354`): 역사 검증을 통과한 템플릿만 스캔에 투입.
- **[6/9] DNA 부검**(`forward/dna_autopsy.py`): 대박주(승자) vs 참사주(패자)의 DNA를 매 리포트 대조 → 진화 방향의 가시화·피드백.
- **둠스데이 γ감쇠**(이전 작업): 위험 국면에서 초신성 켈리 노출 자체를 형상변환 감쇠.

## 5. 정합성·한계 (정직한 평가)

- **기여는 보장, 단 조건부 측정:** 진화 루프(엔진 9·10)는 **템플릿/태그별 청산 표본 ≥ 5건**에서 본격 작동한다. 표본이 적으면 elastic relief로 버티되, 통계적 조정은 보류된다(과적합 방지). 즉 "막 시작한 신규 템플릿"은 즉시 진화하지 않고 표본을 모으는 구간이 있다.
- **하드코딩 base 템플릿 잔존:** `RANK_A~D`(`supernova_hunter.py:1363-1373`)는 고정 시드 벡터다. MFE 황금타점·MULTI 학습 템플릿이 이를 보완·대체하지만, base 자체는 정적이다(콜드스타트 안전판 역할).
- **국고 환수의 의미:** 엔진10의 `CENTRAL_TREASURY += final_balance`는 도태 로직의 회계 처리이며, 이는 별개로 진행 중인 **Live NAV 엔진**(treasury_state.json)과 표기 체계가 다르다. (Live NAV가 실자산 SSOT, CENTRAL_TREASURY는 레거시 도태 회계.)
- **forensics는 섀도우:** 미래 초신성 DNA 매칭(Loop 4)은 `virtual_trade_history`에 쌓이는 무위험 검증이다. 본 장부 자동 편입이 아니라 "정밀도 측정·학습" 용도다.

## 7. 결론

- **Top 1·2·3는 실제 수익에 기여한다.** 일치율 점수 → 켈리 노출액 → `forward_trades` → Live NAV 복리로 직결되며, 성과는 [2/9]·[3/9]·[9/9] 성적표로 환류된다.
- **Top 점수 = 미래 초신성과의 부합도(코사인 일치율)** 그 자체다.
- **부합 안 하고 수익이 안 날 때 미래 초신성과 비슷해지는 진화는 4중으로 구현돼 있다:** 허들 상향(엔진9) → 무능 템플릿 도태·면역(엔진10) → **실전 고MFE 승자 DNA로의 α=0.3 EMA 수렴(엔진6, 핵심)** → 실제 상한가 DNA 섀도우 검증(forensics). 이는 기관급 온라인 러닝(점진 추적 + 백신 + 도태)의 정석 구조다.
- **개선 여지(낮은 우선순위):** ① base RANK 템플릿도 점진 학습 대상에 포함, ② forensics 섀도우 정밀도가 임계 이상이면 본 장부 자동 승격, ③ 도태 회계(CENTRAL_TREASURY)를 Live NAV로 일원화.

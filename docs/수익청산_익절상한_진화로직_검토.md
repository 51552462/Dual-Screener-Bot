# 수익 청산(Exit)·익절 상한·"길게 가져가는 진화" 로직 정밀 검토

> 질문 요약: **"수익률이 아무리 잘 나봤자 ~15%가 끝인 것 같다. 나는 유동적으로 더 길게 가져가는 진화 로직을 만들었는데, 그게 실제로 어떻게 구성돼 있고, 최대 수익률을 계속 끌어올리는 진화형·기관급 퀀트 로직으로 돼 있는지 검토해 달라."**

결론 먼저: **사용자의 직감이 정확합니다.** "길게 가져가는" 장치(강한 종가 오버라이드 · 보유일 연장 · ACE 진화 완화)는 분명히 존재하지만, 그 위에 **익절가를 `진입가 × (1+TP)`로 못 박는 상한 캡**과 **TP를 "승자 MFE의 중앙값(50퍼센타일)"에 묶는 자가튜닝 루프**가 동시에 작동하면서, 구조적으로 **우측 꼬리(대박 러너)를 잘라내고 평균 ~10~11%(특수조건 시 최대 ~19%)에 수익을 수렴**시키고 있습니다. 즉, "최대 수익률을 계속 끌어올리는" 방향이 아니라 **중앙값으로 회귀(regression to the median)**시키는 방향으로 진화하고 있습니다.

---

## 1. 청산 엔진 전체 구조 (`forward/ledger.py`)

매 봉마다 OPEN 포지션을 순회하며 아래 우선순위로 청산을 집행합니다.

| 우선순위 | 트리거 | exit_type | 청산가 산정 | 상한 여부 |
|---|---|---|---|---|
| **1순위-A** | 장중 저가가 MAE 손절선 도달 (`low_ret_pct <= dyn_mae_sl`) | `STAT_MAE` | `ep×(1+MAE/100)` | 손절 고정 |
| **1순위-B** | 장중 고가가 MFE 익절선 도달 (`high_ret_pct >= dyn_mfe_tp`) | `STAT_MFE` | **`ep×(1+TP/100)` ← 상한 캡** | ✅ **익절 상한** |
| 2순위 (STAT) | 보유일 ≥ TIME_STOP 이고 수익<3% | `STAT_TIME` | 종가 | — |
| 2순위 (STAT) | 장중 저가 ≤ ATR 손절가 | `STAT_ATR` | `sl_price` | — |
| 2순위 (TECH/HYBRID) | ZLEMA 추세 이탈(`is_tech_exit`) | `TECH`/`HYBRID_TECH` | **종가 `c` (실제가, 비-상한)** | ❌ 비상한 |
| 3순위 | 보유일 ≥ TIME_STOP×2 (좀비) | `ZOMBIE_FORCE_CLOSE` | `ep` (원금) | — |

최종 실현 수익률은 다음 한 줄로 결정됩니다.

```386:388:forward/ledger.py
                # 💡 [핵심] 최종 수익률(ret)은 희망회로 종가(c)가 아니라 '실제 증권사가 던진 가격(actual_exit_price)' 기반으로 계산
                ret = round(((actual_exit_price - ep) / ep) * 100, 2)
                mfe = round(((new_max - ep) / ep) * 100, 2)
```

→ `mfe`(고점 기준 최대 수익)는 **진짜 고점**을 기록하지만, `final_ret`(실현 수익)은 `actual_exit_price`에 의해 결정됩니다. 그리고 STAT_MFE 경로에서 `actual_exit_price = ep×(1+TP/100)` 이므로, **고점이 +30%를 찍었어도 실현 수익은 TP(≈10~11%)로 깎입니다.**

## 2. "최대 15%" 상한의 정확한 출처

### 2-1. TP(익절선)의 기본값과 증폭

```270:284:forward/ledger.py
            dyn_mfe_tp = safe_float_cast(ns_live_params.get("DYNAMIC_MFE_TP", 10.0), 10.0)
            od_hurdle = safe_float_cast(sys_config.get("DYNAMIC_OD_HURDLE", 20.0), 20.0)
            is_overdrive_on = row_scalar(r, 'v_energy', 0.0) >= od_hurdle
            if is_overdrive_on:
                dyn_mfe_tp *= 1.10
            ...
                if _ace_evo.active:
                    dyn_mfe_tp = ... + _ace_evo.mfe_tp_relax_pct
```

- **기본 TP = 10.0%** (`DYNAMIC_MFE_TP`)
- **오버드라이브**(수급에너지 `v_energy ≥ 20`)면 ×1.10 → **11.0%**
- **ACE 진화 완화**가 활성화되면 최대 **+8.0%p** 추가 → 이론상 최대 **약 19%**

즉 가장 흔한 경로(오버드라이브 OFF, ACE OFF)에서는 **TP = 10%**, 좋은 조건이 다 겹쳐야 **~19%**. 사용자가 체감한 "잘해야 15%"는 이 분포의 중앙~상단입니다.

### 2-2. 익절가를 TP로 못 박는 상한 캡 (핵심 결함)

```332:337:forward/ledger.py
            elif high_ret_pct >= dyn_mfe_tp:
                if c >= l + (h - l) * 0.7:
                    pass
                else:
                    do_exit, exit_rsn, actual_exit_type = True, f"수학적 MFE 장중 도달 익절 ({dyn_mfe_tp}%)", "STAT_MFE"
                    actual_exit_price = ep * (1 + (dyn_mfe_tp / 100.0))
```

- 장중 고가가 TP를 건드린 날:
  - **종가가 당일 봉의 상위 30% 안(`c ≥ l + (h-l)×0.7`)** → `pass` = **청산 안 하고 계속 보유** ("길게 가져가는" 오버라이드).
  - **종가가 약하면** → `STAT_MFE` 청산, 단 **청산가를 TP 수준으로 고정**.
- 문제: 어떤 종목이 강한 종가로 며칠 버텨 +30%까지 갔더라도, **처음으로 약한 종가가 나오는 날** STAT_MFE가 발동하면서 실현가를 `ep×(1+TP/100)` = **+10~11%로 되돌려 깎습니다.** 그 위의 19%p(=30%−11%)는 장부에서 사라집니다(=`final_ret`에 미반영, `mfe`에만 흔적).

> 이 캡은 "TP 지정가 매도 주문이 올라갈 때 체결됐다"는 보수적 모델링입니다. 하지만 동시에 "강한 종가면 계속 보유"라는 재량 오버라이드와 **논리적으로 모순**됩니다(지정가 주문이 있었다면 첫 터치에 이미 체결됨). 결과적으로 **우상향 러너의 실현 수익을 TP로 절단**합니다.

### 2-3. 동일 캡이 평행우주 시뮬레이션에도 전염

`sim_stat_ret`(295행), `sim_breadth_ret`(321행)도 동일하게 약한 종가 시 `dyn_mfe_tp`로 고정됩니다. → **백테스트/실험 장부의 MFE 분포까지 TP에서 절단**되어, 다음 단계 자가튜닝(2-4)의 입력이 오염됩니다.

### 2-4. TP 자가튜닝 루프 — "중앙값 앵커" (자기강화 천장)

`system_auto_pilot.py`의 파라미터 진화부:

```813:816:system_auto_pilot.py

            raw_tp = np.percentile(win_s['mfe_pct'].dropna(), 50) if len(win_s) >= 3 else 10.0
```

```920:932:system_auto_pilot.py
                        new_mfe = float(recent_14['mfe_pct'].mean())
                        old_mfe = float(champ_params.get("DYNAMIC_MFE_TP", new_mfe))
                        ...
                        alpha_smooth = float(min(0.4, (new_obs / max(1, (old_obs + new_obs)))))
                        champ_params["DYNAMIC_MFE_TP"] = round((old_mfe * (1 - alpha_smooth)) + (new_mfe * alpha_smooth), 2)
```

- 신규 TP 후보 = **승자 MFE의 50퍼센타일(중앙값)**.
- 챔피언 TP = 최근 14건 승자 MFE **평균**으로 EWC 스무딩.
- **모멘텀/초신성 전략의 수익은 본질적으로 우측 꼬리(소수의 대박)에서 나오는데, TP를 "중앙값/평균"에 맞추면 TP ≈ 10% 부근에 영구 고착됩니다.** 더해서 2-2의 캡이 큰 승자들의 실현 MFE를 잘라내므로, 다음 튜닝에서 분포가 더 낮아지는 **음의 피드백(천장 자기강화)**이 발생합니다.

> 정리: **TP는 중앙값으로 묶이고(2-4) → 실현가는 그 TP로 절단되며(2-2) → 절단된 결과가 다시 TP 튜닝 입력이 되는(2-3)** 3중 루프가 "최대 수익률을 끌어올리는" 진화를 구조적으로 차단합니다.

## 3. "유동적으로 더 길게 가져가는" 로직 — 실제로 무엇이 있나

사용자가 만든 보유 연장 장치들은 **실재하며 작동**합니다. 다만 모두 "**TP 캡을 우회하는 보조 장치**"일 뿐, "**천장을 올리는 주(主) 엔진**"은 아닙니다.

### 3-1. 강한 종가 오버라이드 (위 2-2)
당일 종가가 봉 상위 30%면 TP 터치에도 청산 보류. → 연속 강세일 동안만 보유 연장.

### 3-2. 홀딩 엣지 스코어 (RL 프록시 / Q-Value 근사)

```345:349:forward/ledger.py
            holding_edge_score = (current_ret_pct / max(1, int(new_bars))) * (
                row_scalar(r, 'v_energy', 1.0) / 10.0
            )
            if holding_edge_score > 1.5:
                opt_time_stop_effective = opt_time_stop_effective + 2
```

- "일당 수익 속도 × 수급 강도"가 높으면 **타임스탑을 +2일 연장**. 추세가 살아있는 종목의 수명을 늘림.
- 단, **타임스탑(2순위)만 연장**할 뿐 **1순위 MFE 캡은 불변**(주석에도 명시).

### 3-3. ACE 진화 오버라이드 (`evolution/ace_exit_bridge.py`)

```119:128:evolution/ace_exit_bridge.py
    extra_days = min(5, max(1, int(round((mult - 1.0) * 12))))
    relax = min(8.0, max(0.5, (mult - 1.0) * 50.0))
    ...
        time_stop_mult=mult,
        min_hold_bars_extra=extra_days,
        mfe_tp_relax_pct=relax,
```

- 에이스 DNA playbook에 일치하는 종목에 한해 **타임스탑 배수↑ + 보유 +최대5일 + TP 완화 +최대8%p**.
- **전제조건**: `ENABLE_ACE_EVOLUTION_WEIGHTING=True` 이고 `ACE_EVOLUTION_FORCE_OBSERVE=False`. 관측 모드면 no-op. → **꺼져 있으면 천장은 10~11% 그대로**.

### 3-4. 청산 모드 결투 (STAT vs TECH vs HYBRID)

```848:850:system_auto_pilot.py
            winner = "TECH" if t_pf > s_pf * 1.1 else "STAT"
            current_config[f"{target_ns}_ACTIVE_EXIT_MODE"] = winner
```

- TECH/HYBRID 모드의 **`is_tech_exit`(ZLEMA 추세 이탈) 청산은 종가(실제가)로 체결** → **비상한**. 추세가 끝까지 살아있다 꺾이는 종목은 이 경로로 +30%도 실현 가능.
- 그러나 **1순위 STAT_MFE 캡이 모드보다 먼저** 평가되므로, "장중 고가 TP 터치 + 약한 종가"가 추세 이탈보다 먼저 오면 결국 TP로 절단됩니다. TECH 청산이 캡을 이기려면 "TP는 터치하되 매일 강한 종가"라는 까다로운 조건이 필요.

### 3-5. 생존 호흡 동기화

```858:860:system_auto_pilot.py

                sync_time_stop = max(3, int(round(avg_bars * 1.2)))
```

- 승자 평균 보유일 ×1.2로 타임스탑 자동 조정 → "잘 버티는 전략은 더 길게". 보유 **기간**의 진화이지 **목표가**의 진화는 아님.

## 4. 종합 진단 — 기관급 관점

| 항목 | 현재 시스템 | 기관급 모멘텀/트렌드 전략 | 판정 |
|---|---|---|---|
| 익절 방식 | **고정 TP**(중앙값 앵커) + 실현가 상한 캡 | **트레일링/래칫**(고점 대비 일정 % 또는 ATR 샹들리에) | ⚠️ 구조적 천장 |
| 우측 꼬리 | TP에서 절단 → 소멸 | 끝까지 추적, 꼬리에서 수익 극대화 | ❌ 꼬리 파괴 |
| 부분 익절 | 없음(전량 단일 청산) | 1차 TP 부분청산 + 나머지 러너 트레일 | ❌ 부재 |
| 목표가 진화 | 중앙값/평균으로 회귀 | 분위수 상단·기대값·트레일폭 자체를 진화 | ⚠️ 천장 자기강화 |
| 보유기간 진화 | ✅ 있음(엣지스코어·ACE·호흡동기화) | 있음 | ✅ 양호 |
| 손절 | ✅ ATR/MAE 동적 | 있음 | ✅ 양호 |

**핵심 결론**: 손절·보유기간·진입 DNA 진화는 정교하지만, **"수익 실현(우측 꼬리)" 차원의 진화가 빠져 있습니다.** "최대 수익률을 계속 끌어올린다"는 목표를 달성하려면, *고정 TP 캡*을 *트레일링 래칫 + 부분 익절 + 꼬리 추종 TP 진화*로 대체하는 것이 정공법입니다.

## 5. 제안 — "최대 수익률 상시 상승" 진화형 청산 아키텍처 (구현 로드맵)

> 아래는 검토 결과에 따른 권고안입니다. (이번 작업은 검토·문서화 단계이며, 코드 변경은 별도 지시 시 진행합니다.)

### 5-1. (즉효) STAT_MFE 캡 → 동적 트레일링 래칫으로 전환
- TP 터치 시 전량 청산하지 말고, **고점 추적 모드(ratchet)** 진입:
  - 트레일 정지선 `trail_stop = max_high × (1 − k)` , 여기서 `k`는 ATR 기반 동적 폭(예: `k = clamp(λ·ATR/price, 4%, 12%)`).
  - 장중 저가가 `trail_stop`을 깨면 **실제가(종가/트레일선)로 청산** → 캡 제거, 우측 꼬리 보존.
- 효과: +30% 러너가 약한 종가 한 번에 11%로 깎이지 않고, 고점 대비 후퇴분만 반납.

### 5-2. 부분 익절(스케일아웃) 도입
- 1차 TP(예: 10%)에서 **포지션 1/3 실현**(시드 회수·심리 안정), 나머지 2/3는 5-1의 트레일로 러너 추종.
- 평균 실현 수익의 분산을 낮추면서 우측 꼬리를 살림.

### 5-3. TP 진화 앵커를 "중앙값 → 꼬리/기대값"으로 교체
- `raw_tp`를 50퍼센타일 대신 **70~80퍼센타일** 또는 **승자 평균 MFE**로 상향, 혹은 TP 자체를 폐기하고 **트레일 폭 `k`를 진화 대상**으로 삼기.
- 자가튜닝 입력에서 **캡되지 않은 `mfe_pct`(고점 기반)**를 쓰도록 보장(2-3 오염 차단).

### 5-4. 추세 가속 시 동적 목표 확장 (이미 있는 엣지스코어 재활용)
- `holding_edge_score > 1.5`일 때 타임스탑만 +2일 늘리는 현 로직을, **트레일 폭 축소 금지 + 부분 재매수(피라미딩)** 트리거로 확장.

### 5-5. ACE 진화 상시화 검토
- `ENABLE_ACE_EVOLUTION_WEIGHTING` 활성 여부·관측 모드 상태를 점검. 꺼져 있으면 3-3의 완화(+8%p)가 무력화되어 천장이 10~11%로 고정됨.

**"길게 가져가는 진화"는 보유기간 차원에서만 작동하고, 수익 실현 차원에서는 '중앙값 고정 TP + 실현가 상한 캡'이 우측 꼬리를 절단해 ~10~15%(특수조건 ~19%)에 수익을 가둡니다. 최대 수익률을 상시 끌어올리려면 고정 TP를 트레일링 래칫·부분 익절·꼬리 추종형 TP 진화로 교체해야 합니다.**

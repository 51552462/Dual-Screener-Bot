# 상승·횡보·하락장(Regime) 판별 — 정밀 검토

> **질문:** "내 퀀트 로직에서 상승/횡보/하락을 **어떻게 구분짓는지** 정밀한 부분이 있는가?"
> **범위:** KR·US 메인 (bitget/legacy 제외) · 코드 변경 없음 · 검토 전용
> **근거 파일:** `regime_meta_analyzer.py`, `meta_governor.py`, `system_auto_pilot.py`, `forward/shared.py`, `meta_state_store.py`

---

## 0. 한 줄 답

**판별 로직은 "정밀하게" 있습니다. 다만 하나가 아니라 서로 다른 임계값을 쓰는 판별기가 3개 공존**하고, 그 셋이 **VIX 기준·추세 기준·대상 지수가 제각각**이라 서로 어긋날 수 있습니다. 거래에 실제로 찍히는 장세 라벨(`entry_regime`)은 그중 **②번(메타거버너 종합)** 입니다.

| # | 판별기 | 입력 | 추세 기준 | VIX 기준 | 산출물 |
|---|--------|------|-----------|----------|--------|
| ① | `regime_meta_analyzer._classify_regime` | ^GSPC·^KS11·^VIX (1개월) | **MA20** (양 지수) | **>20 절대** | `REGIME_ANALYSIS.regime_key`, `CURRENT_REGIME_KEY` |
| ② | `meta_governor._step_regime` (**권위 SSOT**) | ①결과 + bitget + VIX분위 | ①을 계승 | **> 롤링 p90 상대** | `META_REGIME_KEY` → **거래 `entry_regime` 스탬프** |
| ③ | `system_auto_pilot.run_autonomous_analysis` (토 10:00) | SPY·^VIX·RSP (1년) | **EMA200** (SPY) | **<18 / ≥18 / ≥28 절대** | 주간 비중·룩백·`regime_key`(BULL/BEAR/CHOP) |

값: `HIGH_VOL / BULL / BEAR / SIDEWAYS(=CHOP) / UNKNOWN`.

## 1. 판별기 ① — 글로벌 장세 판독기 (SSOT 1차)

**파일:** `regime_meta_analyzer.py` · 함수 `_classify_regime()` L156–224

### 1.1 입력 데이터 (L320–342)
- `yfinance` 로 `^GSPC`(S&P500), `^KS11`(KOSPI), `^VIX` **최근 1개월**(`auto_adjust=True`) 조회. 요청 사이 0.3~1.2초 스텔스 슬립.

### 1.2 지수별 스냅샷 (`_snapshot_index` L128–153)
- `close` = 마지막 종가
- `ma20` = 20일 단순이동평균(`min_periods=10`)
- `above_ma20` = `close > ma20`
- `range5d_pct` = (최근 5일 고가 최댓값 − 5일 저가 최솟값) / 마지막 종가 × 100

### 1.3 임계값 (L53–55)
```
VIX_HIGH          = 20.0   # VIX 절대 임계
NARROW_RANGE_PCT  = 3.5    # 최근 5영업일 고저폭 %
MA_WINDOW         = 20
```

### 1.4 판별 결정트리 (엄격한 우선순위, L165–224)
```
0) SPX·KOSPI 둘 다 조회 실패              → UNKNOWN
1) VIX > 20.0                            → HIGH_VOL   (고변동 발작장)
2) SPX·KOSPI '둘 다' 5일 고저폭 < 3.5%   → SIDEWAYS   (박스·횡보)
3) SPX·KOSPI '둘 다' 종가 > MA20         → BULL       (강세장)
4) SPX·KOSPI '둘 다' 종가 < MA20         → BEAR       (하락·늪지)
5) 그 외(방향 엇갈림/혼재)               → SIDEWAYS   (방향성 탐색)
```

**핵심 성질**
- **BULL/BEAR는 미국·한국이 "둘 다 같은 방향"일 때만** 성립. 한쪽만 위/아래거나 엇갈리면 무조건 **SIDEWAYS**(보수적).
- VIX가 1순위라 **추세가 강해도 VIX>20이면 전부 HIGH_VOL**로 덮어씀.
- 추세 잣대가 **MA20(약 1개월선)** → 단기 추세 판정.

### 1.5 저장·갱신
- `REGIME_ANALYSIS`(regime_key, 라벨, vix, 지수 스냅샷) + `CURRENT_REGIME_KEY` 저장 (L349–372).
- **실행 시점:** 상시 위성이 아니라, `meta_state_store.rebuild_meta_state` 가 REGIME_ANALYSIS 가 없거나 UNKNOWN·지수 ok=0 일 때 **선행 호출**(`meta_state_store.py:484`), 그리고 `factory_artifact_guard.py:196` 자가치유에서 호출. → 사실상 **메타 사이클(일일 prelude)마다 신선화**.

## 2. 판별기 ② — 메타거버너 종합 (권위 SSOT · 실거래 라벨의 실제 출처)

**파일:** `meta_governor.py` · 함수 `_step_regime()` L1140–1171

### 2.1 입력
- ① 의 `REGIME_ANALYSIS.regime_key` (메인 시장)
- bitget `CURRENT_REGIME_KEY` (`_map_bitget_regime` 으로 CHOP/WHIPSAW→SIDEWAYS 정규화)

### 2.2 스트레스 우선 병합 (`_merge_regime_keys` L310–325)
```
HIGH_VOL > BEAR > SIDEWAYS(=CHOP=WHIPSAW) > BULL > UNKNOWN
```
→ 두 출처 중 **더 방어적인 쪽**을 채택. (예: 메인 BULL + bitget BEAR → BEAR)

### 2.3 VIX 동적 승격 (L1147–1164) — ①과 다른 잣대
- `META_VIX_LEVEL_Q` = `_vix_snapshot_quantiles` (최근 **60일 종가**의 분위수, p90 포함; L290–304, 산출 L951).
- **`vix_last > p90` 이면 강제 `HIGH_VOL` 승격**(confidence ≥ 0.86).
- 즉 ②는 **절대 20이 아니라 "최근 60일 대비 상대적으로 높은가(p90 초과)"** 로 고변동을 판정. ①(절대 20)과 **기준이 다름**.
- VIX 오프라인/스킵이면 이 규칙 미적용(노트에 표시).

### 2.4 신뢰도 (L357–365)
- 지수 ok 2개 → 0.82, 1개 → 0.62, 0개 → 0.42/0.25. HIGH_VOL → 0.88/0.55.

### 2.5 산출 → 행동맵 (`ACTION_BY_REGIME` L43–79)
`META_REGIME_KEY` + `META_REGIME_CONFIDENCE` + `META_REGIME_ACTION`:

| 국면 | kelly_cap | S1(공격) 한도 | S4(방어) 한도 |
|------|-----------|---------------|---------------|
| BULL | 0.028 | [1.0, 1.85] | [0.55, 1.15] |
| SIDEWAYS | 0.018 | [0.65, 1.25] | [0.85, 1.45] |
| HIGH_VOL | 0.012 | [0.45, 1.05] | [0.9, 1.55] |
| BEAR | 0.010 | [0.35, 0.95] | [1.05, 1.75] |
| UNKNOWN | 0.015 | [0.55, 1.35] | [0.75, 1.35] |

### 2.6 거래 라벨 스탬프 (★ 가장 중요)
진입(try_add) 시 `forward_trades.entry_regime` 는 **②의 `META_REGIME_KEY`** 로 찍힌다:

```2035:2044:forward/shared.py
                _rk_m = normalize_regime_key(
                    load_meta_state_resolved().get("META_REGIME_KEY")
                )
                cur_regime = (
                    _rk_m
                    if _rk_m not in ("", "UNKNOWN")
                    else resolve_config_regime_key(sys_config)
                )
            except Exception:
                cur_regime = sys_config.get("CURRENT_REGIME_KEY", "UNKNOWN")
```
우선순위: `META_REGIME_KEY`(②) → 없으면 `CURRENT_REGIME_KEY`(①/③) → `UNKNOWN`.

### 2.7 비대칭 기억력 (국면별 룩백, L105–134)
순위·Treasury 통계의 **윈도우 길이**가 국면별로 다름:
```
BEAR / HIGH_VOL → 15~20일 (빠른 방어)
BULL           → 최대 120일 (장기 신뢰)
SIDEWAYS       → 30~60일
```
(주의: 윈도우 "길이"만 국면 의존. 통계를 국면으로 버킷하진 않음 — 별도 문서 `장세별_로직_수익방어_순위_검토.md` 참조.)

## 3. 판별기 ③ — 자율조율 독립 판독기 (토요일 10:00)

**파일:** `system_auto_pilot.run_autonomous_analysis` L400–474, 코어스 매핑 L644

### 3.1 입력 (L421–442) — ①·②와 또 다름
- `yfinance` 로 **SPY·^VIX·RSP 1년치**.
- `spy_ema200` = SPY 200일 EMA(`span=200`).
- **시장 폭(Breadth)** = (현재 RSP/SPY) / (50일 평균 RSP/SPY). RSP=동일비중 S&P, SPY=시총비중 → 비율로 **소수 대형주 쏠림(지수 착시)** 탐지.

### 3.2 결정 (L445–474)
```
SPY > EMA200  AND  VIX < 18      → "Bull (상승장)"        (w1 1.2 / w4 0.8)
그 외                            → "Bear/Chop (하락/횡보)" (w1 0.5 / w4 1.5)

breadth < 0.97 → 취약(쏠림)  → w1 ×0.5, w4 ×1.2
breadth > 1.03 → 강력(확산)  → w1 ×1.2

VIX ≥ 28 → "Bear (극단적 공포장)"  룩백 7,  w1 0 / w4 2
VIX ≥ 18 → (라벨 유지)             룩백 15
VIX < 18 → (라벨 유지)             룩백 45
VIX 스킵 → "Bear/Chop (VIX 스킵)"  룩백 30, w1 0.7 / w4 1.3
```

### 3.3 텍스트 → 키 매핑 (L644) — ⚠️ 미묘한 함정
```python
regime_key = "BULL" if "Bull" in regime else ("BEAR" if "극단적" in regime else "CHOP")
```
| 텍스트 라벨 | 매핑 결과 |
|-------------|-----------|
| "Bull (상승장)" | **BULL** |
| "Bear (극단적 공포장)" | **BEAR** |
| "Bear/Chop (하락/횡보)" | **CHOP** ← 평범한 하락이 CHOP(횡보)로 기록됨 |
| "Bear/Chop (VIX 스킵)" | **CHOP** |

→ **③ 기준으로는 "진짜 BEAR"는 VIX≥28 패닉일 때만.** 일반적 하락은 전부 CHOP(=SIDEWAYS)로 분류된다. (①·②의 BEAR 정의와 불일치)

### 3.4 용도
주간 비중(`WEIGHT_S1/S4`)·동적 룩백·`{regime_key}_CHAMPION_PARAMS` 기억소 로드·국면별 켈리(현재 국면 entry_regime 필터, 엔진 1.9).

## 4. 세 판별기의 불일치 지점 (정밀 검토의 핵심)

| 항목 | ① 판독기 | ② 메타거버너 | ③ 자율조율 |
|------|----------|--------------|-----------|
| 추세 잣대 | **MA20**(SPX+KOSPI) | ① 계승 | **EMA200**(SPY) |
| 고변동 기준 | VIX **>20 절대** | VIX **> 롤링p90 상대** | VIX **≥18/≥28 절대** |
| 대상 시장 | **미국+한국 동시** | 미국+한국+bitget | **미국 단독(SPY/RSP)** |
| BEAR 성립 | 양 지수 < MA20 | 병합 결과 | **VIX≥28일 때만**(평시 하락→CHOP) |
| 횡보 정의 | 5일폭<3.5% or 혼재 | 계승/병합 | "Bear/Chop" 묶음 |

**실무상 위험 포인트**
1. **MA20 vs EMA200 충돌:** 단기선 위·장기선 아래(또는 반대)면 ①은 BULL/SIDEWAYS, ③은 Bear/Chop로 **상반** 가능.
2. **VIX 3중 기준:** 절대 20(①) vs 상대 p90(②) vs 절대 18/28(③). 저변동 국면에선 p90이 18보다 훨씬 낮아 ②가 먼저 HIGH_VOL 승격할 수 있음.
3. **미국 단독 한계(③):** KR만 무너지는 장세를 ③은 못 봄(SPY/RSP만 봄). 반대로 ①은 KR·US 동시 충족을 요구해 **한쪽만 약하면 BULL을 놓침**.
4. **하락의 라벨 표류(③):** 평범한 하락이 CHOP로 기록 → 국면별 통계·기억소가 "BEAR"를 과소표집.
5. **히스테리시스 부재:** ②는 매 사이클 새로 계산(완충은 VIX p90 승격뿐). 임계값 경계에서 **플립플롭(휩쏘)** 여지. 단, 병합이 방어 우선이라 거짓 BULL은 줄지만 과방어 가능.
6. **SIDEWAYS 의미 중첩:** ①에서 "진짜 좁은 박스(폭<3.5%)"와 "방향 혼재"가 **같은 SIDEWAYS** 라벨 → 후속 해석이 두 상황을 구분 못함.
7. **정규화 의존:** `meta_state_store.normalize_regime_key` 가 CHOP/WHIPSAW 등을 SIDEWAYS로 수렴시켜 라벨 표준화(키 불일치 방지).

## 6. 결론

1. **"상승/횡보/하락 구분"은 정밀하게 구현되어 있다.** 추세(MA20/EMA200) + 변동성(VIX) + (③) 시장 폭(breadth)까지 본다.
2. 다만 **단일 SSOT가 아니라 3개 판별기가 서로 다른 임계값으로 공존**한다. 거래 라벨은 **②(META_REGIME_KEY)**, 주간 튜닝은 **③**, 그 ②의 1차 입력은 **①**.
3. **가장 주의할 정밀 이슈:** (a) ③에서 일반 하락이 **CHOP로 표류**, (b) VIX 기준이 **20/p90/18·28** 3중, (c) 추세선이 **MA20 vs EMA200** 로 달라 상반 판정 가능, (d) ①은 **US·KR 동시 합의**를 요구해 한쪽 장세를 놓침.
4. 참고: 외부 지수 의존을 줄이려는 **자체 국면(`weekly_proprietary_regime.py`, `Proprietary_Regime_Audit.md`)** 이 별도로 존재하나, 현재 **공식 SSOT는 위 ①→②** 경로다.

> **개선 후보(구현 시 별도 승인):** ①②③의 VIX·추세 임계값을 단일 설정키로 통일, ③의 "Bear/Chop→CHOP" 매핑을 BEAR/CHOP 분리, 경계 플립 방지용 N일 히스테리시스(연속 X일 동일 조건 시에만 전환) 도입.

## 7. 🧬 전면 재구축 완료 로그 — 예측형 자율 진화 앙상블(Predictive Auto-Evolving Ensemble)

§3~§6 에서 지적한 "3개 판별기 파편화 + 후행성 고정 가중치 + 하락→CHOP 표류 + 휩쏘"를
**단일 스코어링 앙상블 엔진**으로 통합 재구축했다. 신규 모듈: `predictive_regime_ensemble.py`.

### Mission 1 — 확률론적 스코어링 앙상블 (Score-based Ensemble)
- 결정트리를 폐기하고 **5개 독립 팩터**로 분리: `short_trend`(MA20), `long_trend`(EMA200),
  `vix`, `breadth`(RSP/SPY), `pri`(내부 선행).
- 각 팩터는 상태값 `s_f ∈ [-1,+1]` (tanh 정규화) → 가중합 `Σ w_f·s_f` →
  임계(±0.18)로 **BULL/SIDEWAYS/BEAR**, 그리고 **softmax 확률**(`probs`) 동시 산출.
- 결측 팩터는 사용 가중치만 재정규화해 자동 제외(콜드스타트/KR breadth 부재 대응).

### Mission 2 — 가중치 자가 진화 + 탄력적 매크로 닻
- `evolve_weights()`: 관측한 팩터 상태와 **실제 미래 5일 PnL**(`forward_trades`)의 보상
  `reward_f = s_f · tanh(PnL%/scale)` 으로 **skill EMA** 갱신 → softmax 로 가중치 재산출.
- `meta_learner.run_meta_learning_cycle()` 안에서 동반 호출(같은 진화 사이클에 편입).
- **Macro Anchor Floor:** `(long_trend+vix) ≥ 15%` 락, **PRI Cap:** `≤ 85%` —
  `project_weights()` 가 매 갱신마다 반복 투영으로 강제(파멸적 망각 차단). ✅ 검증 통과.

### Mission 3 — 선행 지표 PRI 라이브 편입
- 섀도우 `weekly_proprietary_regime.py` 의 `composite_z` 를 `pri` 팩터로 라이브 편입.
- 외부 지수가 횡보라도 PRI 보상이 누적되면 가중치가 **최대 85%**까지 올라
  '선제적 BULL' 선언이 구조적으로 가능(상한·매크로 닻으로 폭주만 차단).

### Mission 4 — 비대칭 동기화 + 노이즈 필터링

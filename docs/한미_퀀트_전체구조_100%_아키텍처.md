## 1.3 동시성 & 락

| 메커니즘 | 설명 |
|---|---|
| **글로벌 런타임 락** `.factory_runtime.lock` | 모든 factory 모드가 공유하는 단일 flock(4GB 서버 OOM 가드). 내용: `mode\nstarted_at\npid`. Windows=no-op. Stale 자가치유: 죽은 PID/고아파일(>60s)/age>7200s. |
| **순차 단계 실행** `dispatch_factory_mode` | 단계 순서 실행, 각 `run_step` 격리. **좀비 파이프라인 가드**: `critical=True` 단계 실패 시 잔여 단계 전부 skip(스테일 데이터 기반 리포트 방지). |
| **세션 스킵** (2계층) | (1) pre-lock `_scan_session_closed_skip`: 장 닫힘 시 락·prelude 없이 `SKIPPED_SESSION`. (2) 런타임 `require_market_open_for_scan`: KR 09:00–15:30 KST / US 09:30–16:00 ET. override `FACTORY_FORCE_SCAN_OUTSIDE_SESSION=1`. |
| **세션 중복방지** `SessionDeduplicationGuard` | 유효 OPEN 북이 있을 때만 재스캔 차단(표본 가뭄 회복 인지). |
| **슬롯 디스패처** `factory_slot_dispatcher.py` | US용: KST cron으로는 ET 슬롯이 깨지므로, `*/5`분 폴링 → now를 ET로 변환 → `|now−slot|≤4분`이면 실행(DST-safe). `.factory_slot_dispatch_state.json`로 dedup. |
| **워치독** `watchdog.py` | 5분마다(systemd timer) `ops_events.sqlite`의 `heartbeat.tick` 확인. 100s×3회 연속 미스 → 텔레그램 + `systemctl restart dante-factory`. |

### systemd 토폴로지
- `dante-factory.service` → `run_factory_daemon.sh` → `--daemon` (Restart=on-failure).
- `dante-snapshot.timer`(5분) → CQRS `market_data_snapshot.sqlite` 갱신.
- `dante-watchdog.timer`(5분) → `watchdog.py`.
- cron(`/etc/cron.d/dual-screener-factory-{kr,us}`) → 동기 `factory.sh` 파이프라인.

# 4. 듀얼 스크리너 — 스캐닝 파이프라인

## 4.1 두 개의 자본 "암(Arm)" — S1 & S4
"듀얼 스크리너"는 본질적으로 **2-암 자본배분 모델**이다:
- `WEIGHT_S1` — **추세/모멘텀 암**(초신성 + EMA5/눌림 S1 대세추세 448 완전정배열)
- `WEIGHT_S4` — **눌림/바닥탈출 암**(눌림목 S4 텐배거)

기본값 `system_auto_pilot.py`: `WEIGHT_S1=1.0, WEIGHT_S4=1.0, ACCOUNT_SIZE=20,000,000, RISK_PCT=0.02`. 주말 `run_autonomous_analysis`가 VIX/Breadth로 재배분.

사이징 적용(`forward/shared.py`): `sig_type`에 `S1`/`SUPERNOVA` → `×w_s1`, `S4`/`눌림` → `×w_s4`.

## 5.3 청산 엔진 — 우선순위 사다리 (`forward/ledger.py`)
계산: `current_ret/low_ret/high_ret`. 네임스페이스별 파라미터(`{ns}_TIME_STOP`=10, `{ns}_ATR_SL`=2.0, `{ns}_LIVE_PARAMS`→`DYNAMIC_MAE_SL`=-3.5%/`DYNAMIC_MFE_TP`=10.0%). ATR 스탑가 = `ep−(ATR_SL×entry_atr)`.

**Breadth 붕괴 긴급조임**: `<0.97` → time_stop×0.5, 모든 MAE_SL×0.5.

**우선순위 사다리(첫 매칭 승):**
- **P1 절대 MAE/MFE**: `low_ret≤MAE_SL`→`STAT_MAE`; `high_ret≥MFE_TP`→`f_out=fluid_scale_out_fraction()`; f_out≥0.999→`STAT_MFE_FULL` else 부분매도(`scaled_out_frac`, `free_runner=1`)
- **P1b 러너 볼록 트레일링(M2)**: `kappa=convex_ratchet_kappa(run_ret)`, `trail_px=max_high×(1−kappa)`, `l≤trail_px`→`RUNNER_TRAIL`
- **P1c 자가 피라미딩(M4)**: free-runner & edge↑ → `_maybe_pyramid_add()` 자식행
- **RL 타임스탑 연장**: `holding_edge=(ret/bars)×(v_energy/10)>1.5`→time_stop+2; ACE 추가연장
- **P2 모드별**: HYBRID(기본): 타임스탑→`HYBRID_TIME`, ATR→`HYBRID_ATR`, 기술→`HYBRID_TECH`. TECH/STAT 별도
- **P3 좀비 강제청산**: `bars≥time_stop×2`→`ZOMBIE_FORCE_CLOSE`(원금 ep 회수)

## 6.1 전체 Kelly Merge Chain (`forward/shared.try_add`)
하나의 `kelly_risk_pct` 스칼라를 순차 변형:

```
Step0 base = resolve_trading_kelly_base() → regime_kelly_failsafe.resolve_graceful_base_kelly
             (DYNAMIC_KELLY_RISK 기본0.01; UNKNOWN이면 [0.005,0.022] 클램프)
Step1 × WEIGHT_S1 (if S1/SUPERNOVA) × WEIGHT_S4 (if S4/눌림)
Step2 × ts_mult  (네임스페이스 Thompson Beta(α+1,β+1)/0.5, clamp 0.20–1.80)
Step3 × [순환매 2.0][스필오버 1.5][합성생존 1.5]  (incubator는 notional=0)
Step4 apply_meta_kelly_merge:
        × META_GLOBAL_KELLY_MULT
        × doomsday_dampening (1−(max(0,score−40)/60)**γ, γ∈0.5–3.0)
        × META_NS_KELLY_MULT[ns] × META_GROUP_KELLY_MULT[group]
        → regime floor→cap 클램프 (ACTION_BY_REGIME)
        → KILL_SWITCH / 안티패턴 bbox 매칭 → 0
Step5 × resolve_template_multiplier (template_bandit, 0.10–2.00)
Step6 per-logic 복리시드 & 현금브레이크:
        group_seed = ACCOUNT_SIZE(20M) + Σ(group 실현PnL)
        available_cash = group_seed − locked_cash  (≤0 → 거부, 마진 없음)
        raw_shares = (group_seed × kelly)/risk_dist  (리스크패리티)
        sim_kelly_invest = min(raw_shares×ep, group_seed×effective_max_position_pct, available_cash)
```

## 7.1 예측형 자율 진화 앙상블 (`predictive_regime_ensemble.py`)

### 상수
`MACRO_ANCHOR_FLOOR=0.15`, `PRI_WEIGHT_CAP=0.85`, `UP_THRESHOLD=0.18`, `DOWN_THRESHOLD=-0.18`, `SOFTMAX_BETA=6.0`, `VIX_MID=18.0`, `VIX_SCALE=8.0`, `VIX_PANIC_ABS=30.0`, `FWD_EVAL_DAYS=5`, `SKILL_EMA_BETA=0.20`, `PNL_REWARD_SCALE=4.0`, `DEFAULT_HYSTERESIS_DAYS=2`.

### 팩터 상태 (각 `[-1,+1]` tanh)
- short_trend = `tanh(((close/ma20)−1)/0.03)`
- long_trend = `tanh(((close/ma200)−1)/0.08)`

### 판정 & 진화
- score = Σ w_f·s_f (결측팩터 제외 재정규화) → `≥0.18 BULL / ≤−0.18 BEAR / else SIDEWAYS` + softmax 확률

- **Mission 2 안전망** `project_weights`(8회 반복투영): PRI≤0.85(초과 재분배), (long_trend+vix)≥0.15(부족 시 비닻 축소·닻 상향)

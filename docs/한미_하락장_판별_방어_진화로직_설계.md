## 1. 문제 정의 — 왜 하락장에서 주식 퀀트가 어려운가

### 1.1 구조적 한계

| 요소 | 현재 상태 | 하락장 영향 |
|------|-----------|-------------|
| **듀얼 암(S1/S4)** | S1=모멘텀, S4=눌림/바닥 | S1은 BEAR에서 MetaGovernor가 **의도적으로 억제**(weight 상한 0.95) |
| **롱 전용 원장** | `forward_trades` = 가상 롱 | 지수 하락 시 알파만으로 상쇄하기 어려움 |
| **Kelly 복리** | `live_nav_manager` → True NAV | BEAR cap 1.0%로 축소되나, **기존 OPEN**은 청산 엔진에만 의존 |
| **헤지 슬리브** | `inverse_etf_sniper` (테일×30%) | 규모가 작고 OPEN 1건 제한 → **베타 헤지 부족** |
| **숏 엔진** | `blackhole_hunter` (US, 격리 DB) | 메인 Kelly/NAV/Deathmatch와 **미연결** |

### 1.2 목표 재정의

하락장에서 기대하는 성과는 **절대수익 극대화**가 아니라:

1. **MDD 억제** — NAV 고점 대비 낙폭 최소화  
2. **상대 알파** — 지수 대비 초과수익(벤치마크: KOSPI / SPY)  
3. **선택적 수익** — S4 바닥턴, 역축(Dante), 인버스, (확장) 섹터 숏  
4. **회복 탄력** — BULL 전환 시 S1 가중 빠른 복원(히스테리시스 역방향)

### 3.1 SSOT(단일 진실원) 계층

| **2** | `regime_meta_analyzer.py` | `REGIME_ANALYSIS.regime_key` | 앙상블 결과 기록 + 레거시 호환 |

**진입 시 스탬프:** `forward_trades.entry_regime` = `META_REGIME_KEY` (②→③ 경로)

### 3.2 5팩터 앙상블 — BEAR가 선언되는 조건

`predictive_regime_ensemble.py`의 팩터와 임계:

| 팩터 | 입력 | BEAR 방향 신호 |
|------|------|----------------|
| `short_trend` | MA20 대비 종가 | 종가 << MA20 → s_f → −1 |
| `long_trend` | EMA200 대비 | SPY/KOSPI < EMA200 → −1 |
| `vix` | VIX 수준·60일 분위 | VIX↑ → −1 (패닉 ≥30 → KR crisis sync) |
| `breadth` | RSP/SPY vs 50일 평균 | **US만** — 소형주 확산 약화 → −1 |
| `pri` | `weekly_proprietary_regime` composite_z | 내부 통과율·MAE stress → −1 |

**판정식:**  
`score = Σ w_f · s_f` → score ≤ **−0.18** 이면 BEAR (±0.18 사이 SIDEWAYS)

**안전장치:**
- **Macro Anchor Floor:** `(long_trend + vix)` 가중치 합 ≥ 15% (거시 망각 방지)
- **PRI Cap:** ≤ 85% (내부 선행만으로 BULL 오판 방지)
- **히스테리시스:** 기본 2일 연속 동일 후보 → 전환 (RL로 1~5일 자가조절)
- **Kelly Clutch:** 1위 softmax 확률 < 60% → 글로벌 Kelly × 0.10~0.30

### 3.3 보조 판별기 — BEAR를 "더 일찍/더 세게" 잡는 레이어

| 레이어 | 트리거 | BEAR/HIGH_VOL 효과 |
|--------|--------|---------------------|
| **MetaGovernor VIX p90** | VIX > 60일 p90 | HIGH_VOL 강제 승격 |
| **Doomsday DEFCON** | `macro_doomsday_bot` → score ≥70 | DEFCON 1~2: 롱 차단 + INVERSE_MODE |
| **Doomsday Dampener** | global score > 40 | Kelly 기하급수 감쇠 |
| **Breadth Collapse** | RSP/SPY < 0.97 | 청산: time_stop×0.5, MAE_SL×0.5 |
| **Circuit Breaker** | OPEN 손실/시드 ≤ −5% | 신규 진입 차단 (3일 또는 −2% 회복 시 해제) |

### 3.4 BEAR의 3단계 세분화 (현재 암시적 → 명시화 권고)

현재 라벨은 `BULL / SIDEWAYS / BEAR / HIGH_VOL / UNKNOWN` 5종.  
하락 **지속성**과 **수익 전략**을 위해 아래 3단계로 **서브타입**을 두는 것을 권고:

| 서브타입 | 조건 (제안) | 특성 | 대응 전략 |
|----------|-------------|------|-----------|
| **BEAR_GRIND** | score ∈ [−0.18, −0.35], VIX < 25 | 완만한 하락·늪지 | S4↑, 소형 Kelly, Dante 역축 |
| **BEAR_ACCEL** | score < −0.35 또는 20d DD > 8% | 가속 하락 | Kelly cap 추가 50%, scale-out 85% |
| **BEAR_PANIC** | VIX ≥ 30 또는 DEFCON ≤ 2 | 패닉·급락 | 롱 차단, 인버스 풀가동, CB 트립 |

→ 구현 시 `predictive_regime_ensemble`의 `blended_regime` 옆에 `bear_phase` 필드 추가, MetaGovernor `ACTION_BY_REGIME` 확장.
